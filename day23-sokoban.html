📦 Sokoban — Push Every Box Onto a Goal

Push every 📦 onto a 🟩. You can only push boxes — never pull.

Use the arrow keys (or the on-screen pad) to walk the little worker 🧍 around the warehouse. Walk into a box and you'll push it one cell — but only if the square just beyond it is empty. You can never push two boxes at once, and you can never push a box into a wall. The puzzle is solved the instant every goal square has a box sitting on it. The catch: shove a box into a corner it can't come out of and the level becomes impossible — so think before you push.

The board is just a 2D grid of characters — # wall, . goal, $ box, @ worker. Every move looks one cell ahead (and for a push, two cells ahead) to decide what's legal. The Is it still solvable? button runs a breadth-first search over box+worker positions to tell you whether a solution still exists from here. Everything runs offline in vanilla JS.

🔨 Build it yourself

    
A level held as a grid of characters, a "what's at row r, col c?" helper, one move(dr,dc) function that carries the whole push rule, a win check that scans for uncovered goals, and a bit of UI glue. About 90 lines of vanilla JS — copy each block in order.

    
    

      
  1. 
        
    1
    Model the level as a grid
    
        
    A Sokoban level is a rectangle of tiles, so store it as an array of strings — one string per row, one character per cell. The characters are the classic Sokoban alphabet: # wall, space is floor, . goal, $ box, @ worker. Splitting each row string into a 2D array of cells lets you read any tile with grid[r][c]. This one data structure is the entire game state.
    
        
    const LEVEL = [
  "#####",
  "#@$.#",   // worker, box, goal
  "#####",
];
// # wall · (space) floor · . goal · $ box · @ worker
let grid = LEVEL.map(row => row.split(""));  // grid[r][c]
    
      
    2. 

      
  2. 
        
    2
    Separate the moving pieces from the floor
    
        
    Here's the tricky bit: a goal (.) and a box ($) can occupy the same tile — a box parked on its goal. So it's cleaner to keep two layers: a static floor layer that only remembers walls and goals, and dynamic sets for the box positions and the worker position. Then rendering just overlays the movers on top of the floor, and nothing gets lost.
    
        
    let boxes = new Set();          // "r,c" strings
let goals = new Set();
let player = {r:0, c:0};
grid.forEach((row,r) => row.forEach((ch,c) => {
  if (ch === "$") boxes.add(r+","+c);
  if (ch === ".") goals.add(r+","+c);
  if (ch === "@") player = {r, c};
}));
    
      
    3. 

      
  3. 
        
    3
    Turn a key press into a direction
    
        
    Movement is always one step in one of four directions, so map each arrow key to a (dr, dc) delta: up is (-1,0), down is (+1,0), left is (0,-1), right is (0,+1). Row grows downward, so "up" subtracts from the row index. Every move — walking or pushing — funnels through a single move(dr,dc) function, which keeps the rules in one place.
    
        
    const DIRS = {
  ArrowUp:[-1,0], ArrowDown:[1,0],
  ArrowLeft:[0,-1], ArrowRight:[0,1],
};
document.onkeydown = e => {
  const d = DIRS[e.key];
  if (d) { e.preventDefault(); move(d[0], d[1]); }
};
    
      
    4. 

      
  4. 
        
    4
    The push rule — look one (and two) cells ahead
    
        
    This is the heart of Sokoban. Compute the tile in front of the worker. If it's a wall, do nothing. If it's floor, just step there. If it's a box, look one more cell in the same direction: only if that far tile is floor (not a wall, not another box) can you push — move the box forward and step into where it was. Two look-aheads, and the whole game's legality lives right here.
    
        
    function move(dr, dc){
  const nr = player.r + dr, nc = player.c + dc;   // tile ahead
  if (isWall(nr, nc)) return;                     // blocked
  if (boxes.has(nr+","+nc)) {                     // a box is there
    const br = nr + dr, bc = nc + dc;             // tile beyond box
    if (isWall(br, bc) || boxes.has(br+","+bc)) return; // can't push
    boxes.delete(nr+","+nc);                      // push the box
    boxes.add(br+","+bc);
  }
  player = {r:nr, c:nc};                           // worker steps in
  render();  checkWin();
}
    
      
    5. 

      
  5. 
        
    5
    Win check — every goal covered
    
        
    You win when every goal square has a box on it. Since goals never move, the simplest test is: is each goal position also in the box set? Because a level always has exactly as many boxes as goals, "every goal covered" is the same as "every box home". Count the overlap to drive the "boxes home 3/3" readout too.
    
        
    function boxesOnGoal(){
  let n = 0;
  for (const g of goals) if (boxes.has(g)) n++;
  return n;
}
function checkWin(){
  if (boxesOnGoal() === goals.size)
    msg.textContent = "🎉 Solved!";
}

    Render the grid + wire the pad
    
        
    Rendering paints one <div> per tile: draw the floor/wall/goal from the static layer, then overlay a box or the worker on top. A box on a goal gets a "done" colour so progress is visible. Point the on-screen arrow buttons at the same move() the keyboard uses, add a Reset that reloads the level, and the game is complete.
    
        
    function render(){
  board.innerHTML = "";
  board.style.gridTemplateColumns = `repeat(${cols}, 46px)`;
  for (let r=0; r<rows; r++) for (let c=0; c<cols; c++){
    const key = r+","+c, div = document.createElement("div");
    div.className = "cell " + (isWall(r,c) ? "wall"
      : goals.has(key) ? "goal" : "floor");
    if (boxes.has(key)) div.textContent = goals.has(key) ? "✅" : "📦";
    else if (player.r===r && player.c===c) div.textContent = "🧍";
    board.appendChild(div);
  }
}
