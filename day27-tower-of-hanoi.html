🗼 Tower of Hanoi — Move the Whole Stack, One Disk at a Time, Never a Big Disk onto a Small One

Move the whole stack from the left peg to the right peg.

Click a peg to pick up its top disk, then click another peg to drop it — or drag a disk straight across. You may only ever move the topmost disk of a peg, and you may never place a larger disk on top of a smaller one: illegal drops are refused and cost you nothing. The puzzle is solved when every disk sits, largest to smallest, on the right-hand peg. The counter tracks your moves; the best you can possibly do with n disks is 2n−1 moves, and matching it is a perfect game. Stuck? Hit Auto-solve and watch the recursive solution play itself out, disk by disk.

Moves: 0
Optimal (2ⁿ−1): 15
Disks: 4
3 8

The whole game is three arrays of integers — one per peg — where each number is a disk's size and the last item in an array is the disk sitting on top. A move is a single push(pop()) from one array to another; it is legal only when the destination is empty or its top number is bigger than the one you are moving. Auto-solve is the classic recursion in four lines: to move n disks, move the top n−1 onto the spare peg, move the biggest disk across, then move the n−1 back on top — and that is provably the fewest moves possible. All vanilla JavaScript, offline, no images or libraries.

🔨 Build it yourself

    1
    The state: three pegs, each a stack of sizes
    
        
    Each peg is just an array of disk sizes, ordered bottom-to-top, so the last element is the disk on top. A fresh game stacks disks n down to 1 on the source peg — biggest on the bottom, smallest on top — and leaves the other two pegs empty.
    
        
    const SRC = 0, TGT = 2;            // move the stack from peg 0 to peg 2
let N = 4;                         // number of disks
let pegs = [[], [], []];           // pegs[i] = sizes, bottom → top
let moves = 0;

function init(n){
  N = n; moves = 0;
  pegs = [[], [], []];
  for (let s = n; s >= 1; s--) pegs[SRC].push(s);   // n at the bottom … 1 on top
}
const top = p => pegs[p][pegs[p].length - 1];        // the movable disk

    2
    Draw the pegs and the disks
    
        
    Render is a pure function of the three arrays: for each peg, drop one bar per disk with a width that grows with the disk's size, stacked from the bottom up. The peg you have currently picked up from gets its top disk lifted with a highlight. Each peg carries a click handler so a tap knows which peg it is.
    
        
    const board = document.getElementById("hanoi");
function render(){
  board.innerHTML = "";
  for (let i = 0; i < 3; i++){
    const peg = document.createElement("div");
    peg.className = "peg" + (i === TGT ? " target" : "");
    peg.onclick = () => clickPeg(i);
    peg.insertAdjacentHTML("beforeend", '<div class="post"></div><div class="base"></div>');
    pegs[i].forEach((size, idx) => {
      const d = document.createElement("div");
      const isTop = idx === pegs[i].length - 1;
      d.className = "disk" + (held === i && isTop ? " lift" : "");
      d.style.width  = (40 + (size - 1) * (125 / Math.max(1, N - 1))) + "px";
      d.style.background = diskColor(size, N);
      d.textContent = size;
      peg.appendChild(d);
    });
    board.appendChild(peg);
  }
}

    3
    Pick up a disk, then drop it
    
        
    The first click on a non-empty peg picks up its top disk — remember which peg it came from in held. The second click either drops it on another peg (if the move is legal) or, if you click the same peg, puts it back down. Only the topmost disk is ever in play, exactly as the rules demand.
    
        
    let held = null;                       // the peg we picked up from, or null
function clickPeg(i){
  if (solving) return;
  if (held === null){                  // nothing in hand yet
    if (pegs[i].length) { held = i; render(); }      // pick up the top disk
    return;
  }
  if (i === held){ held = null; render(); return; }  // click source again = put back
  if (canMove(held, i)) { doMove(held, i); held = null; afterMove(); }
  else msg("Illegal: you can't put a bigger disk on a smaller one.");
}

    4
    The one rule: never a big disk on a small one
    
        
    Every legal-move check in the whole game is a single line. A move from one peg to another is allowed when the source has a disk to move and the destination is either empty or has a top disk larger than the one arriving. Performing the move is a push(pop()). This one test is what makes an illegal drop cost nothing — it simply never happens.
    
        
    function canMove(from, to){
  if (from === to || !pegs[from].length) return false;
  return pegs[to].length === 0 || top(to) > top(from);   // empty, or bigger on top
}
function doMove(from, to){
  pegs[to].push(pegs[from].pop());     // the whole physics of the game
  moves++;
}

  5. 
        
    5
    Count moves, spot the win, compare to optimal
    
        
    After each move, redraw and check for the win: the puzzle is solved the moment the target peg holds all N disks. Show the running move count next to the theoretical best, 2^N − 1 — matching it is a perfect game, and you can never beat it.
    
        
    const optimal = () => Math.pow(2, N) - 1;      // fewest moves possible
const won     = () => pegs[TGT].length === N;  // all disks home

function afterMove(){
  render();
  document.getElementById("moveLbl").textContent = moves;
  document.getElementById("optLbl").textContent  = optimal();
  if (won())
    msg(moves === optimal() ? `Solved in ${moves} — perfect!` : `Solved in ${moves} moves.`);
}

    6
    The recursive solver — four lines
    
        
    Here is the whole trick. To move n disks from from to to using via as spare: first move the top n−1 onto the spare peg, then move the single biggest disk to the target, then move those n−1 from the spare onto the target. It calls itself twice on a smaller problem; the base case is doing nothing when n hits zero. Instead of touching the pegs directly, it records each move so the game can replay them.
    
        
    function solve(n, from, to, via, out){
  if (n === 0) return;                 // base case: no disks, no moves
  solve(n - 1, from, via, to, out);    // 1. top n-1 : from → via
  out.push([from, to]);                // 2. biggest disk : from → to
  solve(n - 1, via, to, from, out);    // 3. those n-1 : via → to
}
// usage: const list = []; solve(N, 0, 2, 1, list);
// list.length === 2**N - 1, and every move in it is legal.

    Animate the solution move by move
    
        
    Auto-solve resets to a clean start, asks solve for the full list of moves, then plays them on a timer — apply one move, redraw, repeat — so you watch the disks migrate. Faster stacks get a slower tick so a 3-disk solve is easy to follow while an 8-disk solve (255 moves) still finishes in a few seconds. A second click stops it.
    
        
    let solving = false, timer = null;
function autoSolve(){
  if (solving){ clearInterval(timer); solving = false; return; }
  init(N); held = null; render();
  const list = []; solve(N, SRC, TGT, 3 - SRC - TGT, list);
  const dt = Math.min(600, Math.max(45, Math.floor(9000 / list.length)));
  let k = 0; solving = true;
  timer = setInterval(() => {
    if (k >= list.length){ clearInterval(timer); solving = false; afterMove(); return; }
    const [f, t] = list[k++]; doMove(f, t); render();
  }, dt);
}

    Reset and choose the disk count
    
        
    A slider from 3 to 8 sets how many disks you play with; changing it just re-initialises and redraws. Reset does the same for the current count. Because every rule is a tiny operation over three arrays, the whole difficulty curve is one number.
    
        
    function reset(){ init(N); held = null; afterMove(); }

document.getElementById("diskRange").oninput = e => {
  if (solving) return;
  init(+e.target.value); held = null; afterMove();
  document.getElementById("diskLbl").textContent = N;
};
document.getElementById("solveBtn").onclick = autoSolve;
document.getElementById("resetBtn").onclick = reset;
init(4); afterMove();
