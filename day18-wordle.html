🟩 Wordle — Guess the Word in Six

Guess the hidden five-letter word in six tries — the tiles tell you how close you are.

Type any five-letter word and press Enter. Each tile then flips to reveal a colour: green means that letter is correct and in the right spot, yellow means the letter is in the word but somewhere else, and gray means the letter isn't in the word at all. Use those clues to narrow it down — the on-screen keyboard remembers everything you've learned by colouring its keys. You have six guesses; nail the word to win, or run out and the answer is revealed.

Duplicate letters are handled the real Wordle way: a two-pass scan marks the exact-position greens first, then hands out yellows only for the letters the answer still has left over — so guessing a word with a repeated letter never lights up more copies than the answer actually contains. Only valid five-letter words from the built-in list are accepted; anything else shakes the row. Everything runs offline in plain vanilla JS — both the on-screen keys and your physical keyboard work.

🔨 Build it yourself

    
A 6×5 grid of tiles, a current-row buffer that captures typing, the two-pass scoring algorithm that colours each letter green / yellow / gray (handling duplicates correctly), a CSS flip reveal, an on-screen keyboard whose keys take the best colour they've earned, win/lose detection, and a new-game that picks a random answer. ~90 lines of vanilla JS — copy each block in order.

    
    

      
  1. 
        
    1
    Build the 6×5 board
    
        
    Wordle is a fixed grid: six guesses tall, five letters wide. We build it once as 30 empty tile elements arranged in 6 rows of 5, and keep a 2D array of references so we can find any tile by row and column later. The board never grows or shrinks — guesses just fill the next empty row. CSS grid does the layout; JavaScript only fills in letters and colours.
    
        
    
          
    const ROWS = 6, COLS = 5;
const board = document.getElementById("board");
const tiles = [];                          // tiles[r][c] = element
for (let r = 0; r < ROWS; r++){
  const row = document.createElement("div");
  row.className = "row";
  tiles[r] = [];
  for (let c = 0; c < COLS; c++){
    const t = document.createElement("div");
    t.className = "tile";
    row.appendChild(t);
    tiles[r][c] = t;                       // remember each tile
  }
  board.appendChild(row);
}
    
          
        
    
      
    2. 

      
  2. 
        
    2
    Capture typing into the active row
    
        
    Two numbers track where we are: row (which guess) and col (how many letters typed so far). A letter key writes into the current tile and moves col right (capped at 5). Backspace moves col left and clears the tile. Enter submits. We listen on the physical keyboard and route the on-screen keys through the same handleKey function so both inputs behave identically.
    
        
    
          
    let row = 0, col = 0;
let current = "";                          // letters typed in this row

function handleKey(key){
  if (over) return;
  if (key === "ENTER")      submitGuess();
  else if (key === "BACK"){ if (col > 0){ col--; current = current.slice(0,-1); tiles[row][col].textContent = ""; tiles[row][col].classList.remove("filled"); } }
  else if (/^[A-Z]$/.test(key) && col < COLS){
    tiles[row][col].textContent = key;
    tiles[row][col].classList.add("filled");
    current += key; col++;
  }
}
document.addEventListener("keydown", e => {
  if (e.key === "Enter") handleKey("ENTER");
  else if (e.key === "Backspace") handleKey("BACK");
  else if (/^[a-zA-Z]$/.test(e.key)) handleKey(e.key.toUpperCase());
});
    
          
        
    
      
    3. 

      
  3. 
        
    3
    The two-pass scoring algorithm
    
        
    This is the heart of Wordle and the part most people get wrong. You can't colour in one pass, because duplicate letters need careful counting. Pass 1: walk the guess and mark every letter that's in the exact right spot green — and decrement a tally of the answer's remaining letters. Pass 2: walk again over the not-yet-green letters; colour one yellow only if the answer still has that letter left over in the tally, then decrement. Anything left is gray. This guarantees a repeated guess letter never gets more colours than the answer actually has copies of.
    
        
    
          
    function score(guess, answer){
  const result = Array(COLS).fill("gray");
  const left = {};                                  // unused answer letters
  for (const ch of answer) left[ch] = (left[ch]||0) + 1;

  // PASS 1 — greens (exact position), consume from the tally
  for (let i = 0; i < COLS; i++){
    if (guess[i] === answer[i]){ result[i] = "green"; left[guess[i]]--; }
  }
  // PASS 2 — yellows, only if a copy is still left over
  for (let i = 0; i < COLS; i++){
    if (result[i] === "green") continue;
    if (left[guess[i]] > 0){ result[i] = "yellow"; left[guess[i]]--; }
  }
  return result;                                    // e.g. ["green","gray","yellow",...]
}

  4. 
        
    4
    Reveal with the flip + colour the keyboard
    
        
    With the colours computed, we paint them one tile at a time with a small stagger so the row "rolls over" via the CSS flip animation. Half-way through each tile's flip (when it's edge-on) we apply its colour class, so the colour appears as it turns. After a tile is coloured we upgrade its keyboard key — but only ever to a better colour: green beats yellow beats gray, so a key that already went green can never drop back to yellow.
    
        
    
          
    const RANK = { gray:0, yellow:1, green:2 };       // precedence

function reveal(guess, result){
  result.forEach((color, i) => {
    const t = tiles[row][i];
    setTimeout(() => {
      t.classList.add("reveal");
      setTimeout(() => t.classList.add(color), 270);  // colour mid-flip
      paintKey(guess[i], color);                       // upgrade the key
    }, i * 280);
  });
}
function paintKey(letter, color){
  const k = keyEls[letter];
  if (RANK[color] > RANK[keyState[letter] || "gray0"]) {  // only upgrade
    keyState[letter] = color;
    k.classList.remove("green","yellow","gray");
    k.classList.add(color);
  }
}
    
          
        
    
      
    5. 

      
  5. 
        
    5
    Validate the guess on submit
    
        
    When Enter is pressed we guard the submission before scoring. The guess must be exactly five letters long, and it must be a real word from our built-in list — otherwise we shake the row and bail out, leaving the player to fix it. Only a valid guess flows through to scoring, revealing, and the win/lose check. Keeping all the rejection logic up front means the rest of the pipeline can assume it's always handling a legal word.
    
        
    
          
    function submitGuess(){
  if (current.length < COLS) return shake("Not enough letters");
  const guess = current.toLowerCase();
  if (!WORDS.includes(guess)) return shake("Not in word list");

  const result = score(guess, answer);
  reveal(guess, result);
  finish(guess, result);                 // win / lose / next row
}
function shake(text){
  msg.textContent = text;
  tiles[row][0].parentElement.classList.add("shake");
  setTimeout(() => tiles[row][0].parentElement.classList.remove("shake"), 400);
}
    
          
        
    
      
    6. 

      
  6. 
        
    6
    Win / lose detection
    
        
    A guess wins the instant it equals the answer. We check that right after revealing; on a match we set a game-over flag and show a victory message. If it wasn't the answer, we advance to the next row and reset the column. If that was the sixth and final row, the game is lost — so we reveal the answer to the player. The single over flag stops all further typing once the game ends, win or lose.
    
        
    
          
    let over = false;

function finish(guess, result){
  if (guess === answer){
    over = true;
    setTimeout(() => msg.textContent = "🎉 You got it!", COLS * 280);
  } else if (row === ROWS - 1){
    over = true;                                  // used all 6 rows
    setTimeout(() => msg.textContent = "The word was: " + answer.toUpperCase(), COLS * 280);
  } else {
    row++; col = 0; current = "";                 // move to next guess
  }
}

    7
    Pick a random answer + new game
    
        
    A new game picks a random word from the answer list, then wipes the board to a clean slate: clear every tile's text and colour classes, reset the row/col counters and the current buffer, clear the game-over flag, and recolour the keyboard back to neutral. Real Wordle uses one shared daily word — to get that, seed the index from today's date instead of Math.random() so everyone gets the same puzzle. Double-click the file to play; no server needed.
    
        
    
          
    const ANSWERS = ["crane","slate","ghost","plumb","vivid",/* ...~30 */];
let answer;

function newGame(){
  answer = ANSWERS[Math.floor(Math.random() * ANSWERS.length)];
  // daily mode instead:  let i = daysSinceEpoch() % ANSWERS.length;
  row = 0; col = 0; current = ""; over = false; msg.textContent = "";
  for (let r = 0; r < ROWS; r++) for (let c = 0; c < COLS; c++){
    const t = tiles[r][c];
    t.textContent = "";
    t.className = "tile";                          // strip filled + colours
  }
  keyState = {};
  Object.values(keyEls).forEach(k => k.classList.remove("green","yellow","gray"));
}
newGame();
