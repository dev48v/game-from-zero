🃏 Memory Match — Find the Pairs

Sixteen face-down cards hide eight matching pairs — flip two at a time and remember where everything is.

Every card is face-down at the start. Click one to flip it and reveal its emoji, then flip a second. If the two faces match, the pair locks face-up and stays put. If they don't, both cards quietly flip back down after a moment — so your memory is the only thing keeping score of where each picture was. Clear all eight pairs in as few moves and as little time as you can, then try to beat your best.

Memory Match
Flip two cards at a time. Match all eight pairs to win.

A move is one pair of flips. Input locks for a beat while a mismatched pair flips back, so a fast double-click can't break the game. The deck is shuffled with Fisher-Yates every round, so the layout is different each time. Everything runs offline with plain DOM, a CSS 3D flip, and one setTimeout; on touch screens a tap flips just like a click.

🔨 Build it yourself

    
A 4×4 grid of cards, a deck built from eight emoji pairs and shuffled with Fisher-Yates, a CSS 3D flip on each card, a click handler that tracks your first and second pick, a match check that locks or flips back, an input lock during the flip-back, win detection, and a moves/timer/best score. ~80 lines of vanilla JS — copy each block in order.

    
    

      
  1. 
        
    1
    Build the deck and shuffle it
    
        
    Take eight emoji, double the list so every picture appears twice, and you have a 16-card deck of matching pairs. To make each round unpredictable we shuffle it with the Fisher-Yates algorithm: walk from the last card backwards, and swap each one with a random card at or before its position. That gives a perfectly fair, unbiased shuffle in a single pass — far better than the lopsided Math.random() - 0.5 sort trick you'll see online.
    
        
    
          
    const FACES = ["🍎","🚀","🐶","🌺","⚽","🎸","🍕","⭐"];
function buildDeck(){
  const deck = [...FACES, ...FACES];          // 8 pairs → 16 cards
  for (let i = deck.length - 1; i > 0; i--){  // Fisher-Yates shuffle
    const j = Math.floor(Math.random() * (i + 1));
    [deck[i], deck[j]] = [deck[j], deck[i]];  // swap
  }
  return deck;
}

    2
    Render the 4×4 grid of cards
    
        
    Each card is two stacked faces inside a flipper: a green back you see at the start and an emoji front that's hidden until you flip. CSS grid lays the 16 cards out four-by-four. We store the card's value and its index in dataset so the click handler can identify exactly what was clicked. The array of card elements becomes our handle on the whole board.
    
        
    
          
    const board = document.getElementById("board");
const cards = [];
buildDeck().forEach((value, i) => {
  const card = document.createElement("div");
  card.className = "card";
  card.dataset.value = value;            // its emoji
  card.dataset.index = i;                // its slot
  card.innerHTML = `<div class="card-inner">
      <div class="card-face card-back">❓</div>
      <div class="card-face card-front">${value}</div>
    </div>`;
  board.appendChild(card);
  cards.push(card);
});
    
          
        
    
      
    3. 

      
  3. 
        
    3
    Flip a card on click
    
        
    A single click listener on each card does the flipping. But it must refuse bad clicks: ignore the card if the board is locked (a pair is mid-check), if it's already flipped, or if it's already matched. When the click is valid we add the flipped class — the CSS rotates the card 180° to show its emoji — and then route it into the first-pick / second-pick logic.
    
        
    
          
    cards.forEach(card => card.addEventListener("click", () => {
  if (locked) return;                                  // mid-check
  if (card.classList.contains("flipped")) return;      // already up
  if (card.classList.contains("matched")) return;      // already solved
  card.classList.add("flipped");                       // CSS does the 3D turn
  pick(card);
}));
    
          
        
    
      
    4. 

      
  4. 
        
    4
    Track first pick vs second pick
    
        
    Memory Match is a two-step interaction, so we hold the first flipped card in a variable. If nothing is held yet, this click becomes the first pick and we just wait. If a first pick already exists, this click is the second — we count one move and run the match check. After each pair is resolved we clear the first pick back to null, ready for the next turn.
    
        
    
          
    let first = null;          // the first card of the current pair
let moves = 0;

function pick(card){
  if (!first){
    first = card;          // remember it, wait for the second flip
    return;
  }
  moves++;                 // a "move" = one pair of flips
  movesEl.textContent = moves;
  checkMatch(first, card);
}
    
          
        
    
      
    5. 

      
  5. 
        
    5
    Match → lock, or mismatch → flip back (with input lock)
    
        
    Comparing two cards is just comparing their stored values. If they match, we mark both matched so they stay face-up forever and bump the pair count. If they don't, we set locked = true to freeze the board, wait ~800 ms so the player can memorise the faces, then flip both back down and unlock. That input lock is the single most important guard — without it a quick third click would flip a new card while two are still showing and corrupt the game.
    
        
    
          
    let locked = false;

function checkMatch(a, b){
  if (a.dataset.value === b.dataset.value){     // same emoji → match
    a.classList.add("matched");
    b.classList.add("matched");
    matched++; pairsEl.textContent = matched;
    first = null;
    if (matched === 8) win();
  } else {                                       // no match → flip back
    locked = true;                               // freeze all input
    setTimeout(() => {
      a.classList.remove("flipped");
      b.classList.remove("flipped");
      first = null;
      locked = false;                            // unfreeze
    }, 800);
  }
}

    6
    The move counter and timer
    
        
    Two simple scores tell the story of a run. The move counter (already bumped on every second flip) measures how efficiently you played. The timer is a setInterval that ticks once a second, started on the very first flip rather than on page load so the clock only counts real play. We keep its id so we can stop it the moment you win.
    
        
    
          
    let seconds = 0, clock = null;
function startClock(){
  if (clock) return;                  // already running
  clock = setInterval(() => {
    seconds++;
    timeEl.textContent = seconds;
  }, 1000);
}
// call startClock() inside the very first flip
    
          
        
    
      
    7. 

      
  7. 
        
    7
    Win detection and best score
    
        
    The board is solved the instant all eight pairs are matched, which we already check after every successful pair. On a win we stop the clock, show the overlay with your moves and time, and compare the run to the best stored in localStorage. We rank a run by moves first (then time as a tie-break) and save it only if it beats the record, so your best survives a refresh.
    
        
    
          
    function win(){
  clearInterval(clock); clock = null;
  const better = !best || moves < best.moves ||
                 (moves === best.moves && seconds < best.time);
  if (better){
    best = { moves, time: seconds };
    localStorage.setItem("mmBest", JSON.stringify(best));
  }
  bestEl.textContent = best.moves + " moves";
  showOverlay("You win! 🎉", `${moves} moves · ${seconds}s`);
}
    
          
        
    
      
    8. 

      
  8. 
        
    8
    Shuffle & restart
    
        
    Restarting is a clean reset: stop the clock, zero out moves, pairs, time and the first-pick, unlock the board, then wipe and rebuild the grid from a freshly shuffled deck so the layout is brand new. Because we tear down and re-render the cards, there are no stale flipped/matched classes to clean up. Double-click the file to play; no server needed.
    
        
    
          
    function restart(){
  clearInterval(clock); clock = null;
  first = null; locked = false;
  moves = 0; matched = 0; seconds = 0;
  movesEl.textContent = 0; pairsEl.textContent = 0; timeEl.textContent = 0;
  board.innerHTML = "";               // remove old cards
  cards.length = 0;
  renderDeck(buildDeck());            // fresh shuffle, fresh grid
  overlay.classList.add("hidden");
}
