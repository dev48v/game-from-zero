💀 Hangman — Guess the Hidden Word Letter by Letter Before the Drawing Is Complete

Uncover the word before the figure hangs.

The computer has picked a secret word and shown you only its length — a row of blanks. Guess one letter at a time, either by tapping the on-screen keys or just typing on your keyboard. If the letter is in the word, every place it appears is revealed at once and the key turns green. If it is not, you lose a life and one more piece of the poor stick figure is drawn: first the head, then the body, both arms, both legs. Six wrong guesses and the drawing is complete — game over. Fill in every blank before that happens and you have saved him. A one-line hint nudges you toward the category, and your win streak keeps count across rounds. There is no timer and no penalty for thinking, so this is pure deduction: spend your early guesses on common vowels and letters, then use the shape of the word to close it out.

Won: 0
Win streak: 0
Lives left: 6
Wrong: 0
Hint: —
Guess a letter — tap a key below or use your keyboard.

This is real Hangman logic, nothing faked. The whole game is driven by three tiny pieces of state: the secret word, a Set of letters you have guessed, and a single wrong-guess counter. The blanks you see are not stored anywhere — they are rebuilt from scratch on every render by walking the secret word and asking, letter by letter, "is this one in the guessed set?" A correct guess adds a letter to that set; a wrong one bumps the counter. The gallows drawing is a pure function of that one counter — head at one mistake, body at two, arms at three and four, legs at five and six — so the picture can never disagree with the score. You win the instant no blank remains, and lose the instant the counter hits six. All vanilla JavaScript on one canvas, offline, no images or libraries.

🔨 Build it yourself

    
A word list, a guessed Set that dedupes for free, a single wrong-guess counter that doubles as your lives, a reveal that falls straight out of set membership, a win test that asks whether any blank is left, and a gallows drawing gated entirely on that one counter. About 90 lines of vanilla JavaScript on one canvas — copy each block in order.

    
    

      
  1. 
        
    1
    The word list and the game state
    
        
    A Hangman round is defined by remarkably little: the secret word (with a hint), the set of letters tried so far, and one wrong counter. The limit of six wrong guesses is the whole difficulty knob — one for each body part. Everything the player sees is derived from these few variables.
    
        
    const WORDS = [
  { word:"PLANET",    hint:"Earth is one of eight" },
  { word:"CANVAS",    hint:"Where we draw the gallows" },
  { word:"ALGORITHM", hint:"A precise step-by-step recipe" },
  // …add as many as you like
];
const MAX_WRONG = 6;                 // six wrong guesses = full figure
let secret, guessed, wrong, state;   // the entire game state

function newGame(){
  secret = WORDS[Math.random()*WORDS.length | 0];   // pick one at random
}

  2. 
        
    2
    The guessed Set and one way in
    
        
    Every letter the player tries lands in a Set, not an array. A Set dedupes automatically, so guessing "E" twice is a harmless no-op, and it answers "have we tried this?" in one call. Both good and bad guesses live in the same set — whether a letter counts as wrong is decided by asking the word, not by extra bookkeeping.
    
        
    function newGame(){
  /* …pick secret… */
  guessed = new Set();     // letters tried, deduped for free
  wrong   = 0;
  state   = "playing";
}
function guess(letter){
  letter = letter.toUpperCase();
  if (state !== "playing" || guessed.has(letter)) return;  // ignore repeats
  guessed.add(letter);
  if (!secret.word.includes(letter)) wrong++;   // a wrong letter costs a life
  settle();                                      // decide what happens next
}
    
      
    3. 

      
  3. 
        
    3
    Win, lose, or keep playing
    
        
    After each guess, one function settles the round. You have won when no blank remains — every letter of the secret is in the set. You have lost when the wrong counter reaches the limit. Otherwise the state stays "playing". A win bumps a streak; a loss resets it to zero.
    
        
    function isWon(){
  return secret.word.split("").every(ch => guessed.has(ch));  // no blanks left
}
let wins = 0, streak = 0;
function settle(){
  if (isWon())                 { state = "won";  wins++; streak++; }
  else if (wrong >= MAX_WRONG) { state = "lost"; streak = 0;      }
  render();                                      // repaint everything
}
    
      
    4. 

      
  4. 
        
    4
    The reveal (blanks fall out of the Set)
    
        
    Drawing the word is a single map over the secret. For each character, if it is in the guessed set we show it; otherwise a blank. We never mutate the word or track positions — the reveal is a pure read of set membership. On a loss we relax the test so the missed letters show too.
    
        
    function renderWord(){
  const el = document.getElementById("word");
  el.innerHTML = secret.word.split("").map(ch => {
    const show = guessed.has(ch) || state === "lost";   // reveal all on a loss
    return `<span class="wchar">${show ? ch : ""}</span>`;
  }).join("");
  document.getElementById("hint").textContent = "Hint: " + secret.hint;
}

  5. 
        
    5
    The A–Z keyboard, colour-coded
    
        
    Build one button per letter of the alphabet. After a key is used it is coloured by asking the word: green if the letter was in it, grey if it was a wasted guess. Because the colour is derived from state, a full re-render always shows the correct thing without any manual toggling.
    
        
    const ALPHA = "ABCDEFGHIJKLMNOPQRSTUVWXYZ".split("");
function buildKeyboard(){
  const kb = document.getElementById("keyboard");
  kb.innerHTML = "";
  ALPHA.forEach(l => {
    const b = document.createElement("button");
    b.textContent = l;  b.onclick = () => guess(l);
    kb.appendChild(b);
  });
}
function keyClass(l){                    // colour a key from state alone
  if (!guessed.has(l))         return "kbtn kb-idle";
  if (secret.word.includes(l)) return "kbtn kb-hit";   // green: it was in the word
  return "kbtn kb-miss";                                // grey: a wasted guess
}

    6
    Draw the gallows from one number
    
        
    The whole scene is a function of the wrong counter. The gallows is always drawn; then each body part is gated behind a threshold — head at 1, torso at 2, arms at 3 and 4, legs at 5 and 6. Because the picture reads only wrong, it can never disagree with the score.
    
        
    const ctx = document.getElementById("gallows").getContext("2d");
const line = (x1,y1,x2,y2) => { ctx.beginPath(); ctx.moveTo(x1,y1); ctx.lineTo(x2,y2); ctx.stroke(); };

function drawFigure(){
  ctx.clearRect(0,0,280,320);
  ctx.strokeStyle = "#64748b"; ctx.lineWidth = 6; ctx.lineCap = "round";
  line(30,305,150,305); line(70,305,70,35); line(70,35,190,35); line(190,35,190,68); // gallows
  ctx.strokeStyle = state === "lost" ? "#f43f5e" : "#e2e8f0"; ctx.lineWidth = 5;
  if (wrong >= 1){ ctx.beginPath(); ctx.arc(190,92,22,0,Math.PI*2); ctx.stroke(); }   // head
  if (wrong >= 2) line(190,114,190,190);   // body
  if (wrong >= 3) line(190,135,160,165);   // left arm
  if (wrong >= 4) line(190,135,220,165);   // right arm
  if (wrong >= 5) line(190,190,165,235);   // left leg
  if (wrong >= 6) line(190,190,215,235);   // right leg
}

    7
    Two inputs, one door
    
        
    A letter can arrive from a click on an on-screen key or a press on the physical keyboard, but both funnel into the same guess() function. That is the payoff of keeping the rules in one place: the input layer only normalises the letter and hands it over; the deduping, life-spending, and win/lose check all happen inside.
    
        
    buildKeyboard();     // on-screen keys wired to guess(l) in step 5
document.addEventListener("keydown", e => {
  if (/^[a-zA-Z]$/.test(e.key))                      guess(e.key);   // physical key
  else if (e.key === "Enter" && state !== "playing") newGame();     // Enter restarts
});

  8. 
        
    8
    The scoreboard (lives are derived)
    
        
    The HUD reads straight from state. Notice there is no separate "lives" variable to keep in sync — lives is simply the limit minus the wrong count, computed when shown. The message line just reports the outcome, revealing the word on either a win or a loss.
    
        
    function hud(){
  document.getElementById("livesStat").textContent  = MAX_WRONG - wrong;   // derived
  document.getElementById("wrongStat").textContent   = wrong;
  document.getElementById("winStat").textContent     = wins;
  document.getElementById("streakStat").textContent  = streak;
}
function message(){
  if (state === "won")  msg(`Saved! The word was ${secret.word}. 🎉`, "win");
  if (state === "lost") msg(`Out of guesses — it was ${secret.word}.`, "lose");
}

    9
    Render, reset, and boot
    
        
    One render() repaints everything from state, so nothing can drift out of sync. Starting a new word is just resetting the three variables and rebuilding the keyboard. Call newGame() once at the bottom and the game is live.
    
        
    function render(){ drawFigure(); renderWord(); updateKeys(); hud(); message(); }
function newGame(){
  secret  = WORDS[Math.random()*WORDS.length | 0];
  guessed = new Set();  wrong = 0;  state = "playing";
  buildKeyboard();  render();
}
newGame();                       // deal the first word and start playing
