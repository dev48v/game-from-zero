🧩 Nonogram — Draw the Hidden Picture from Number Clues

The numbers describe a hidden picture. Fill the grid to reveal it.

Each number beside a row or above a column is the length of a run of filled cells in that line, listed in order, with at least one gap between runs. A row clue of 2 1 means "a block of two filled cells, then a gap, then one filled cell." Left-click a cell to fill it (emerald); right-click to mark it as a definite empty (an ×). A clue turns green with a ✓ the moment your line matches it. Solve every row and every column at once and the picture is done. Every "New puzzle" is built from a real hidden bitmap, so the clues always have a solution.

Under the hood the puzzle is a hidden grid of 1s and 0s. The clues are computed by scanning each row and column for runs of consecutive 1s — that single run-length function produces every number you see. Winning is not pixel-matching: the game recomputes the run-lengths of your filled cells and checks that every row clue and every column clue is satisfied, which is the true Nonogram condition. Everything runs offline in vanilla JavaScript — no images, no server.

🔨 Build it yourself

    
A hidden grid of 1s and 0s, one run-length function that turns any line into its clue, a renderer that draws the grid with clue gutters, click handlers for fill and ×, a win check that matches your filled cells against the clues, and a generator that starts from a real picture so the puzzle is always solvable. About 110 lines of vanilla JS — copy each block in order.

    
    

      
  1. 
        
    1
    Two grids and three cell states
    
        
    Keep the hidden picture in one grid and what the player has drawn in another. The solution holds 1 (filled) or 0 (empty). The player grid needs a third value because "I know this is empty" is different from "I have not decided": use 0 unknown, 1 filled, 2 marked-empty (×). The clues shown to the player are derived only from the solution.
    
        
    const N = 5;
let solution = [];   // solution[r][c] = 1 filled / 0 empty  (hidden)
let player   = [];   // player[r][c]   = 0 unknown / 1 filled / 2 mark-×

function blank(){ return Array.from({length:N}, () => Array(N).fill(0)); }
    
      
    2. 

      
  2. 
        
    2
    Turn a line into its run-length clue
    
        
    A clue is just the lengths of the consecutive runs of filled cells, in order. Scan the line, count a run, and push its length whenever the run ends. An all-empty line has the clue [0]. Run it over every row to get the row clues and over every column to get the column clues — one function, both directions.
    
        
    function lineClue(line){        // line = array of 0/1
  const runs = []; let run = 0;
  for (const v of line){
    if (v === 1) run++;
    else if (run){ runs.push(run); run = 0; }
  }
  if (run) runs.push(run);
  return runs.length ? runs : [0];   // empty line → [0]
}
const rowClues = () => solution.map(row => lineClue(row));
const colClues = () => Array.from({length:N}, (_,c) =>
  lineClue(solution.map(row => row[c])));

    3
    Draw the grid with clue gutters
    
        
    Lay it out as four quadrants: an empty corner, column clues across the top, row clues down the left, and the N×N cells. Give the clue gutters and the cell grid the same fixed cell size so their tracks line up perfectly. Each cell's class comes straight from the player state.
    
        
    function render(){
  grid.style.gridTemplateColumns = `repeat(${N}, var(--cs))`;
  grid.innerHTML = "";
  for (let r=0;r<N;r++) for (let c=0;c<N;c++){
    const d = document.createElement("div");
    d.className = "ncell " + (player[r][c]===1?"fill":player[r][c]===2?"mark":"");
    d.textContent = player[r][c]===2 ? "×" : "";
    d.oncontextmenu = e => { e.preventDefault(); mark(r,c); };
    d.onclick       = ()=> fill(r,c);
    grid.appendChild(d);
  }
  // colclues / rowclues rendered the same way, one box per line
}

  4. 
        
    4
    Left-click fills, right-click marks ×
    
        
    A left-click toggles between filled and unknown; a right-click toggles between × and unknown. Both re-render and re-check after every change. Preventing the browser context menu on right-click is what lets you use it for marking empties.
    
        
    function fill(r,c){ player[r][c] = player[r][c]===1 ? 0 : 1; update(); }
function mark(r,c){ player[r][c] = player[r][c]===2 ? 0 : 2; update(); }
function update(){ render(); checkWin(); }

  5. 
        
    5
    Win = your filled cells match every clue
    
        
    Do not compare pixels to the hidden grid. Recompute the run-lengths of the player's filled cells (treat × and unknown as empty) and check they equal the clue for every row and every column. That is the real Nonogram win condition, and it correctly accepts any layout that satisfies the clues. The same per-line test drives the green ✓ on each solved clue.
    
        
    const asBits = line => line.map(v => v===1 ? 1 : 0);
function lineOK(line, clue){
  const g = lineClue(asBits(line));
  return g.length===clue.length && g.every((v,i)=>v===clue[i]);
}
function checkWin(){
  const rc = rowClues(), cc = colClues();
  const rowsOK = rc.every((cl,r)=>lineOK(player[r], cl));
  const colsOK = cc.every((cl,c)=>lineOK(player.map(row=>row[c]), cl));
  return rowsOK && colsOK;   // 🎉 picture complete
}

    Generate from a real picture
    
        
    Never invent clues directly — start from a grid. Randomly fill a bitmap (or draw a shape), then compute its clues; because the picture itself satisfies them, the puzzle is solvable by construction. Reject the degenerate all-empty and all-full boards. For guaranteed unique puzzles you would run a solver on the clues and keep only pictures it recovers exactly.
    
        
    function generate(density = 0.55){
  do {
    solution = Array.from({length:N}, () =>
      Array.from({length:N}, () => Math.random()<density ? 1 : 0));
  } while (!hasSomeFilledAndSomeEmpty(solution));
  player = blank();
  render();
}
