💡 Lights Out — Switch Off Every Light

Turn every light off. Clicking a tile flips it — and its four neighbours.

Every tile is a light that is either on (green) or off (dark). Click any tile and it flips: on becomes off, off becomes on — but so do the tiles directly above, below, left and right of it. That coupling is the whole puzzle: you can never touch a single light on its own, so switching one off often flicks its neighbours back on. The board is solved the instant all lights are off. Because every "New puzzle" is built by starting from an all-off board and applying random clicks, a solution is always guaranteed to exist — and the Solve button will find it.

Grid

The board is a grid of 0/1 bits. A click XORs a fixed "plus-shaped" pattern (the tile and its four neighbours) into that bit-grid, and the win test is simply "are all bits zero?". New puzzle generates by applying random clicks to a solved board, so it is always solvable. Show a solution uses light chasing — sweep top to bottom pressing any lit tile from the row above, then correct the top row using a small lookup — the classic linear-algebra-over-GF(2) trick. Everything runs offline in vanilla JS.

🔨 Build it yourself

    
A grid of 0/1 bits, one press(r,c) that XORs a plus-shape into the grid, a win check that asks "are all bits zero?", a generator that scrambles a solved board with random presses, and a light-chasing solver. About 90 lines of vanilla JS — copy each block in order.

    
    

      
  1. 
        
    1
    Model the board as a grid of bits
    
        
    Every tile is a light that is either on or off, so the whole board is just an N×N grid of 1s and 0s. Store it as a 2D array where grid[r][c] is 1 for a lit tile and 0 for a dark one. Because the only two states are on and off, "flip a light" is exactly the XOR operation bit ^= 1. That single data structure is the entire game state.
    
        
    const N = 5;
let grid = Array.from({length:N}, () =>
  Array(N).fill(0));           // grid[r][c] = 1 (on) or 0 (off)

function flip(r, c){ grid[r][c] ^= 1; }   // on↔off is just XOR 1
    
      
    2. 

      
  2. 
        
    2
    A press flips a tile AND its four neighbours
    
        
    This is the rule that makes Lights Out. Pressing tile (r,c) flips five lights in a plus shape: the tile itself plus up, down, left and right — anything off the edge is simply skipped. You never touch a light in isolation, which is what makes the puzzle a puzzle. All the game logic lives in this one function.
    
        
    const DELTAS = [[0,0],[-1,0],[1,0],[0,-1],[0,1]]; // self + 4 neighbours
function press(r, c){
  for (const [dr,dc] of DELTAS){
    const nr = r+dr, nc = c+dc;
    if (nr>=0 && nr<N && nc>=0 && nc<N) grid[nr][nc] ^= 1;
  }
}
    
      
    3. 

      
  3. 
        
    3
    Win check — are all the lights off?
    
        
    The board is solved the instant every light is off. Since off is stored as 0, the whole win test is "does any cell still hold a 1?". Scan the grid; if you find no lit tile, the puzzle is done. Run this after every press.
    
        
    function isSolved(){
  for (let r=0; r<N; r++)
    for (let c=0; c<N; c++)
      if (grid[r][c]) return false;   // a light is still on
  return true;                        // every light is off 🎉
}
    
      
    4. 

      
  4. 
        
    4
    Generate a puzzle that is always solvable
    
        
    Do NOT fill the grid with random bits — most random boards are unsolvable. Instead start from a fully-solved (all-off) board and apply a handful of random presses. Whatever those presses did, pressing the same tiles again undoes it, so a solution is guaranteed to exist. This is the safe way to make a scramble.
    
        
    function generate(k = 8){
  grid = Array.from({length:N}, () => Array(N).fill(0)); // solved
  for (let i=0; i<k; i++){
    const r = (Math.random()*N)|0, c = (Math.random()*N)|0;
    press(r, c);            // random legal moves keep it solvable
  }
}
    
      
    5. 

      
  5. 
        
    5
    Light chasing — clear everything but the top row
    
        
    Here is the classic solve. Sweep from the second row down: whenever the tile directly above is lit, press the current tile to turn it off. After one full top-to-bottom pass, every row except the very top is guaranteed dark — each press was forced, so there is nothing to think about.
    
        
    function chase(){
  const moves = [];
  for (let r=1; r<N; r++)
    for (let c=0; c<N; c++)
      if (grid[r-1][c]){        // light above is on
        press(r, c);            // press here to kill it
        moves.push([r, c]);
      }
  return moves;   // now only the top row may still be lit
}
    
      
    6. 

      
  6. 
        
    6
    Pick the right top-row presses, then chase
    
        
    After a chase only the top row can still be lit, and exactly which top-row tiles to press first is what finishes the job. There are only 2N possible top-row press combinations (32 for a 5-wide board), so just try each one: press that combination, chase down, and keep whichever leaves the whole board dark. Point the on-screen Solve button at this and Lights Out is complete.
    
        
    function solve(){
  for (let mask=0; mask < (1<<N); mask++){
    const g = grid.map(row => row.slice());   // work on a copy
    for (let c=0; c<N; c++)
      if (mask & (1<<c)) pressOn(g, 0, c);    // try these top presses
    chaseOn(g);                               // then sweep down
    if (allOff(g)) return mask;               // this one cleared it
  }
  return null;   // (never happens for a generated board)
}
