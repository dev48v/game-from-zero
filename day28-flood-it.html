🌊 Flood-It — Flood the Whole Board to One Colour in the Fewest Moves

Turn the whole board into a single colour.

Your territory starts as the little region in the top-left corner (marked with a dot). Every time you pick a colour from the palette below, your entire territory repaints to that colour — and any neighbouring cells that already had that colour get swallowed into it. So each move floods outward, absorbing more of the board. Chain the right colours and the whole grid becomes one colour. Do it within the move limit and you win. Stuck? Tap Hint for the greedy best move, or Auto-solve to watch a greedy solver flood the board for you.

The whole game is one 2-D array of colour numbers. Your territory is the connected clump of same-coloured cells touching the top-left corner — found with a flood-fill (BFS/DFS from the origin). A move recolours exactly that clump to a new colour, after which it fuses with adjacent cells of the new colour, so the next flood reaches further. You win when a single flood-fill covers all size² cells — the board is one region. The Hint uses a greedy rule: try each colour and keep the one that grows your territory the most. Greedy is a strong, cheap heuristic — but not always the true optimum (finding the absolute fewest moves is a hard search problem). All vanilla JavaScript, offline, no images or libraries.

🔨 Build it yourself

    
One 2-D array of colour numbers, a seeded random board, a renderer that paints each cell, a flood-fill that finds the connected region touching the top-left corner, a move that recolours that region, a win test that asks whether one flood covers everything, and a greedy hint that tries every colour and keeps the biggest gain.     The state: the board is one 2-D array of colour ids
    
        
    The entire game is a grid of small integers — each cell holds a colour index into a palette. SIZE is the side length, COLORS how many colours are in play, and a palette maps each id to an actual CSS colour. Your territory always starts at the top-left corner [0][0].
    
        
    const PALETTE = ["#ef4444","#f97316","#eab308","#22c55e","#3b82f6","#a855f7","#ec4899","#14b8a6"];
let SIZE   = 14;     // board is SIZE x SIZE
let COLORS = 6;      // how many colours (2..8)
let grid   = [];     // grid[r][c] = colour id 0..COLORS-1
let moves  = 0;
const ORIGIN = [0, 0];             // your territory grows from here
const inB = (r,c) => r>=0 && r<SIZE && c>=0 && c<SIZE;

    A random board — with a seed so it's repeatable
    
        
    Fill every cell with a random colour id. A tiny seeded generator (mulberry32) means the same seed always produces the same board, so a "New game" is just a new seed, and you can share a hard board by its number.
    
        
    function mulberry32(a){ return function(){ a|=0; a=a+0x6D2B79F5|0;
  let t=Math.imul(a^a>>>15,1|a); t=t+Math.imul(t^t>>>7,61|t)^t;
  return ((t^t>>>14)>>>0)/4294967296; }; }

function newGame(seed){
  const rand = mulberry32(seed);
  grid = []; moves = 0;
  for (let r=0; r<SIZE; r++){
    const row = [];
    for (let c=0; c<SIZE; c++) row.push(Math.floor(rand()*COLORS));
    grid.push(row);
  }
}

  4. 
        
    4
    Flood-fill: find the region touching the corner
    
        
    Your territory is every cell you can reach from [0][0] by stepping between orthogonal neighbours that share the corner's colour — a classic flood-fill / breadth-first search. Push the origin, then repeatedly pop a cell and enqueue its same-coloured, unvisited neighbours. The visited set is the territory.
    
        
    const DIRS = [[1,0],[-1,0],[0,1],[0,-1]];
function originRegion(){
  const target = grid[0][0];
  const seen = Array.from({length:SIZE}, () => Array(SIZE).fill(false));
  const stack = [[0,0]]; seen[0][0] = true;
  const cells = [[0,0]];
  while (stack.length){
    const [r,c] = stack.pop();
    for (const [dr,dc] of DIRS){
      const nr=r+dr, nc=c+dc;
      if (inB(nr,nc) && !seen[nr][nc] && grid[nr][nc]===target){
        seen[nr][nc]=true; stack.push([nr,nc]); cells.push([nr,nc]);
      }
    }
  }
  return cells;   // exactly the connected origin region
}

  5. 
        
    5
    A move: recolour the region, absorb neighbours
    
        
    Picking a colour repaints every cell of your current region to it. Choosing the colour you already have does nothing (it's not a move). After the repaint, cells adjacent to your region that already had the new colour are now the same colour as you — so the next flood-fill reaches them automatically. That's how the territory grows.
    
        
    function play(newColor){
  if (newColor === grid[0][0]) return false;   // no-op, not a move
  const region = originRegion();
  for (const [r,c] of region) grid[r][c] = newColor;
  moves++;
  return true;      // adjacent same-colour cells fold in on the next flood
}

    6
    Win / lose: one region, within the limit
    
        
    You've won the instant the board is a single colour — equivalently, when one flood-fill from the corner covers all SIZE² cells. The move limit scales with board size and colour count; we give a small buffer over what the greedy solver needs so every board is winnable, but tight enough to be a real challenge.
    
        
    const won   = () => originRegion().length === SIZE*SIZE;   // whole board = 1 region
let limit = 0;
function computeLimit(greedyEstimate){
  // tuned by board size AND colour count, floored to stay beatable
  limit = Math.max(Math.round(SIZE * COLORS / 3), greedyEstimate + 2);
}
function afterMove(){
  render();
  if (won())            msg(`Solved in ${moves} moves!`);
  else if (moves >= limit) msg(`Out of moves — try again.`);
}

    The greedy hint — grow the territory the most
    
        
    A cheap, strong heuristic: for each colour, make a copy of the board, flood with that colour, and measure the resulting territory. Keep the colour that grows it most. It's greedy — it maximises the immediate gain each turn. Run it to completion and you get a good move count to aim below.
    
        
    function greedyBest(){
  const cur = grid[0][0]; let best=-1, bestSize=-1;
  for (let col=0; col<COLORS; col++){
    if (col === cur) continue;
    const copy = grid.map(row => row.slice());
    for (const [r,c] of originRegionOf(copy)) copy[r][c] = col;
    const size = originRegionOf(copy).length;   // territory after flooding
    if (size > bestSize){ bestSize = size; best = col; }
  }
  return best;                       // the colour with the biggest gain
}
