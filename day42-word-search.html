🔤 Word Search — Hunt Eight Hidden Words Buried in a 12×12 Grid of Random Letters, Placed Across, Down and Diagonally, Forwards and Backwards; Drag a Straight Line of Cells to Grab a Word, Watch It Highlight Green and Cross Off the List, and Race the Clock to Clear the Board

Eight words are hiding in the noise. Drag a line through each one.

A word search is a lattice of letters with real words tucked inside it — running left-to-right, top-to-bottom, on either diagonal, and sometimes backwards. The whole grid is generated fresh each round: eight words are dropped into a 12×12 board along one of eight directions, allowed to cross each other only where their letters agree (exactly like a crossword), and every leftover cell is stuffed with a random letter as camouflage. To grab a word, click-drag a straight line of cells from its first letter to its last — or tap the start, then tap the end on touch. The game reads the letters along your line forwards and backwards, checks them against the list, and if it’s a hit the cells lock green and the word gets struck off. Clear all eight to win. No images, no engine — just a 2-D array of characters, a placement algorithm, and a little vector geometry to turn two cells into a line.

Everything here is a couple of loops. 🔨 Build it yourself

    
There is no sprite sheet and no word database beyond a small array — the whole game is a 2-D array of letters plus a pinch of vector geometry. Building the puzzle is two loops: drop eight words into the grid along one of eight directions (overlapping only where letters already agree), then fill every empty cell with a random letter to hide them. Reading a selection is the same walk in reverse — two clicked cells define a step vector, and the cells between are start + step × i. Validating is a string compare forwards and backwards. About 120 lines of vanilla JavaScript, no libraries. Copy each block in order.

    The board is a 2-D array of letters
    
        
    No canvas, no tiles — the entire puzzle is an N×N array where every cell holds one character. We start it completely empty (blank strings) so the placement step can tell “free” from “taken.” A parallel list, placements, remembers each word we hide and the exact cells it occupies, so later we can check a selection against the truth instead of guessing.
    
        
    const N         = 12;   // grid is 12 × 12
const NUM_WORDS = 8;    // words hidden per game

let grid;         // grid[r][c] = one letter (or "" while building)
let placements;   // [{ word, cells:[{r,c}...], found:false }]

function blankGrid(){
  return Array.from({length:N}, () => Array(N).fill(""));
}

    2
    Eight directions are eight step vectors
    
        
    A word can run in any of eight compass directions, and each one is just a (row step, col step) pair. East is “column +1, row unchanged”; south-west is “row +1, column −1”; and so on. Once a direction is a vector, walking a word across the grid is one loop — add the vector to the current cell to get the next. The four negative directions are what give you backwards and upward words.
    
        
    const DIRS = [
  [0, 1], [0,-1],   // E,  W
  [1, 0], [-1,0],   // S,  N
  [1, 1], [1,-1],   // SE, SW
  [-1,1], [-1,-1]   // NE, NW
];
// cell i of a word = { r: r0 + dr*i, c: c0 + dc*i }

    3
    Place one word — try, check, commit
    
        
    To hide a word we gamble: pick a random start cell and a random direction, then check whether it fits. Two things can go wrong — the word runs off the edge of the board, or it collides with a letter already there. So we first bounds-check the last cell, then walk the word cell by cell. Only if the whole path is legal do we actually write the letters; otherwise we throw the attempt away and roll again, up to a couple hundred times.
    
        
    function tryPlace(word){
  for (let attempt = 0; attempt < 200; attempt++){
    const [dr,dc] = DIRS[rand(DIRS.length)];
    const r0 = rand(N), c0 = rand(N);
    const rEnd = r0 + dr*(word.length-1);
    const cEnd = c0 + dc*(word.length-1);
    if (rEnd<0 || rEnd>=N || cEnd<0 || cEnd>=N) continue;  // off-board
    if (fits(word, r0, c0, dr, dc)) return commit(word, r0, c0, dr, dc);
  }
  return null;   // gave up on this word
}

  4. 
        
    4
    Words may cross only where letters agree
    
        
    This is the one rule that makes a word search feel dense instead of sparse. As we walk the candidate path, each cell is fine if it is empty, or if it already holds the exact letter this word wants there — that’s a shared crossing, just like a crossword. Any other occupied cell is a conflict and the whole placement is rejected. Commit simply writes each letter and records the cell path for later.
    
        
    function fits(word, r0, c0, dr, dc){
  for (let i = 0; i < word.length; i++){
    const cell = grid[r0+dr*i][c0+dc*i];
    if (cell !== "" && cell !== word[i]) return false;  // clash
  }
  return true;                                          // free or matching
}
function commit(word, r0, c0, dr, dc){
  const cells = [];
  for (let i = 0; i < word.length; i++){
    const r = r0+dr*i, c = c0+dc*i;
    grid[r][c] = word[i];
    cells.push({r, c});
  }
  return cells;
}

    5
    Generate the whole puzzle
    
        
    Now assemble a game: shuffle a pool of candidate words, try to place them one by one until eight have stuck, and record each success in placements. A word that can’t find room after 200 tries is skipped and the next pool word is tried instead. Finally, sweep the board and fill every still-empty cell with a random letter — that camouflage is what turns a few scattered words into a puzzle.
    
        
    function newGame(){
  grid = blankGrid();
  placements = [];
  for (const word of shuffle([...POOL])){
    if (placements.length >= NUM_WORDS) break;
    const cells = tryPlace(word);
    if (cells) placements.push({ word, cells, found:false });
  }
  for (let r = 0; r < N; r++)
    for (let c = 0; c < N; c++)
      if (grid[r][c] === "") grid[r][c] = LETTERS[rand(26)];  // fill blanks
  render();
}

    The classic search algorithm
    
        
    This is the algorithm a solver (or our Hint button) uses to find a word without knowing where it was placed: stand on every cell, and from each one try all eight directions, matching the target letter by letter and bailing the instant a character or a boundary disagrees. If a whole direction matches, you’ve found the word and can return its cell path. It is a brute-force scan — rows × cols × 8 × word length — but on a 12×12 board that is nothing.
    
        
    function findWord(word){
  for (let r = 0; r < N; r++)
    for (let c = 0; c < N; c++)
      for (const [dr,dc] of DIRS){
        let ok = true;
        for (let i = 0; i < word.length; i++){
          const rr = r+dr*i, cc = c+dc*i;
          if (rr<0||rr>=N||cc<0||cc>=N || grid[rr][cc] !== word[i]){ ok=false; break; }
        }
        if (ok){                                  // matched in this direction
          const cells = [];
          for (let i = 0; i < word.length; i++) cells.push({r:r+dr*i, c:c+dc*i});
          return cells;
        }
      }
  return null;
}

    Turn two cells into a straight line
    
        
    A selection is defined by just two cells: where you started and where you are now. That pair is only a legal word-line if it is horizontal, vertical, or a perfect 45° diagonal — i.e. the row change is zero, the column change is zero, or their absolute values are equal. When it qualifies, the number of steps is the larger delta, the per-step direction is the sign of each delta, and the cells between are the same start + step × i walk we used to place words.
    
        
    function lineBetween(a, b){
  const dr = b.r - a.r, dc = b.c - a.c;
  const straight = (dr === 0 || dc === 0 || Math.abs(dr) === Math.abs(dc));
  if (!straight) return [a];                       // not a valid line yet
  const len = Math.max(Math.abs(dr), Math.abs(dc));
  const sr = Math.sign(dr), sc = Math.sign(dc);
  const path = [];
  for (let i = 0; i <= len; i++) path.push({ r:a.r+sr*i, c:a.c+sc*i });
  return path;
}

    Drag to select with pointer events
    
        
    One set of pointer handlers covers mouse and touch. pointerdown on a cell sets the anchor; while the button is held, pointermove uses elementFromPoint to find whichever cell is under the finger and recomputes the line from anchor to there, repainting the preview; pointerup ends the drag and commits. Because a tap is a down-then-up with no move, a one-cell result is routed to the tap-start / tap-end path instead — free touch support.
    
        
    grid.addEventListener("pointerdown", e => {
  const cell = e.target.closest("[data-r]"); if (!cell) return;
  e.preventDefault();
  selecting = true; dragMoved = false;
  start = pos(cell); selPath = [start]; paint();
});
document.addEventListener("pointermove", e => {
  if (!selecting) return;
  const el = document.elementFromPoint(e.clientX, e.clientY);
  const cell = el && el.closest && el.closest("[data-r]"); if (!cell) return;
  const end = pos(cell);
  if (end.r !== start.r || end.c !== start.c) dragMoved = true;
  selPath = lineBetween(start, end); paint();
});
document.addEventListener("pointerup", () => { if (selecting){ selecting=false; commit(); } });

    Validate a selection against the words
    
        
    When a line is committed, check it against the truth we stored at generation time. Because a word reads the same to a player whether they swept it left-to-right or right-to-left, a selection matches a placement if its cell sequence equals the placement’s cells or that list reversed. Matching cell paths (not just letters) means a lucky line through random filler can never falsely trigger a word. A hit flips found, locks the cells green, and strikes the word off.
    
        
    function samePath(a, b){                    // b, forwards or reversed
  if (a.length !== b.length) return false;
  const fwd = a.every((p,i) => p.r===b[i].r          && p.c===b[i].c);
  const rev = a.every((p,i) => p.r===b[b.length-1-i].r && p.c===b[b.length-1-i].c);
  return fwd || rev;
}
function evaluate(path){
  if (path.length < 2) return;
  const hit = placements.find(p => !p.found && samePath(path, p.cells));
  if (!hit){ msg("Not a word — try again."); return; }
  hit.found = true;
  hit.cells.forEach(p => cellEls[p.r][p.c].classList.add("found"));
  renderWords();                            // re-draws the struck-off list
  if (placements.every(p => p.found)) win();
}

    Draw the grid, strike the list, win
    
        
    Rendering is pure state. The grid is a CSS grid of N columns; each cell is a div tagged with its data-r/data-c so a pointer can name it, and its classes (sel while dragging, found once caught) come straight from the game state. The word list re-renders on every find, adding a done class that line-throughs the caught words. When placements.every(found) is true the timer stops and the win message appears — New game rebuilds a fresh board from scratch.
    
        
    function render(){
  gridEl.style.gridTemplateColumns = `repeat(${N}, 1fr)`;
  gridEl.innerHTML = ""; cellEls = [];
  for (let r = 0; r < N; r++){
    cellEls[r] = [];
    for (let c = 0; c < N; c++){
      const d = document.createElement("div");
      d.className = "ws-cell"; d.textContent = grid[r][c];
      d.dataset.r = r; d.dataset.c = c;
      gridEl.appendChild(d); cellEls[r][c] = d;
    }
  }
  renderWords();
}
function win(){ stopTimer(); msg("🎉 Solved! All words found."); }
