🦠 Game of Life — Simple Rules, Endless Emergence

Four tiny rules on a grid of cells produce gliders, oscillators and endless surprise.

This is Conway's Game of Life. Every green square is a live cell, every white one is dead. There are no players — you set up a starting pattern, press Play, and each generation every cell looks at its eight neighbours and follows the same rules at once: a live cell with 2 or 3 live neighbours survives, otherwise it dies; a dead cell with exactly 3 live neighbours is born. Click or drag on the grid to toggle cells, use Step to advance one generation at a time, Randomize to sprinkle life, and the preset buttons to drop in famous patterns. This grid is toroidal — the edges wrap around, so a glider that leaves the right side reappears on the left.

The rules are B3/S23: a cell is Born if it has exactly 3 live neighbours, and it Survives if it has 2 or 3 — every other cell ends up dead. Each generation is computed into a fresh buffer and then swapped in, so every cell decides its next state from the same snapshot of the current generation. Everything runs offline in plain vanilla JS — clicking, dragging, presets, and the toroidal wrap-around all work.

🔨 Build it yourself

    
A grid held as a flat array of 0/1 cells, a render that draws a CSS-grid of squares, a neighbours(r,c) counter that looks at the eight surrounding cells (wrapping at the edges), a step() that computes the next generation into a separate buffer using the B3/S23 rules and then swaps it in, click/drag toggling, a play loop driven by setInterval at an adjustable speed, and a handful of preset patterns. ~90 lines of vanilla JS — copy each block in order.

    
    

      
  1. 
        
    1
    Model the grid as a flat array
    
        
    The whole universe is one array of ROWS × COLS numbers, where 0 means a dead cell and 1 means alive. Cell index i maps to row Math.floor(i/COLS) and column i%COLS. Keeping the state flat makes copying and scanning simple loops, and lets us allocate a matching "next generation" buffer in one line. We only translate an index into row and column coordinates when we count neighbours or draw a cell.
    
        
    
          
    const COLS = 40, ROWS = 30, CELLS = COLS * ROWS;
let grid = new Uint8Array(CELLS);      // 0 = dead, 1 = alive

const rowOf = i => Math.floor(i / COLS);
const colOf = i => i % COLS;
const idx   = (r, c) => r * COLS + c;  // (row,col) → flat index

  2. 
        
    2
    Render the grid of cells
    
        
    We build one small <div> per cell once, arranged by a CSS grid with 40 columns. On every render we read the array and toggle an alive class, which paints the cell green or leaves it white. Re-rendering just re-reads the array, so what you see and the data behind it never drift apart. Building the DOM once and only flipping classes keeps redraws fast even at 30 generations a second.
    
        
    
          
    const board = document.getElementById("board");
board.style.gridTemplateColumns = `repeat(${COLS}, 15px)`;
const cellEls = [];
for (let i = 0; i < CELLS; i++){
  const el = document.createElement("div");
  el.className = "cell"; el.dataset.i = i;
  board.appendChild(el); cellEls.push(el);
}
function render(){
  for (let i = 0; i < CELLS; i++)
    cellEls[i].classList.toggle("alive", grid[i] === 1);
}
    
          
        
    
      
    3. 

      
  3. 
        
    3
    Count the eight neighbours (with wrap)
    
        
    Every cell has eight neighbours — up, down, left, right, and the four diagonals. We loop over the nine offsets from −1 to +1 and skip the centre (the cell itself). To make the grid toroidal, we wrap coordinates with modular arithmetic: adding ROWS/COLS before the % keeps the result positive, so a cell on the top edge counts the bottom edge as its neighbour. The count returned is a number from 0 to 8 — the single input every rule depends on.
    
        
    
          
    function neighbours(g, r, c){
  let n = 0;
  for (let dr = -1; dr <= 1; dr++)
    for (let dc = -1; dc <= 1; dc++){
      if (dr === 0 && dc === 0) continue;      // skip self
      const rr = (r + dr + ROWS) % ROWS;        // wrap rows
      const cc = (c + dc + COLS) % COLS;        // wrap cols
      n += g[idx(rr, cc)];
    }
  return n;                                     // 0..8
}

  4. 
        
    4
    The four rules → the B3/S23 test
    
        
    Conway's four rules collapse into two conditions. A live cell survives only with 2 or 3 neighbours — fewer is underpopulation (it dies), more is overpopulation (it dies). A dead cell becomes alive only with exactly 3 neighbours (reproduction). Written compactly this is "Born on 3, Survives on 2 or 3", or B3/S23. This one expression is the entire law of the universe — everything else is just applying it to every cell.
    
        
    
          
    // given a cell's current state and its neighbour count,
// return its next state (0 or 1) under B3/S23
function nextState(alive, n){
  if (alive) return (n === 2 || n === 3) ? 1 : 0;   // survive
  return (n === 3) ? 1 : 0;                          // birth
}

    5
    Compute the next generation into a buffer
    
        
    This is the crux. We allocate a brand-new array for the next generation and fill it by reading the current grid. Every cell decides its fate from the same untouched snapshot, then we swap the buffer in as the new grid. If we mutated cells in place, a cell we already updated would corrupt the neighbour count of the next one, and the simulation would be wrong. Buffer-then-swap (double buffering) keeps all cells stepping simultaneously — the whole point of a cellular automaton.
    
        
    
          
    function step(){
  const next = new Uint8Array(CELLS);   // fresh buffer
  for (let i = 0; i < CELLS; i++){
    const n = neighbours(grid, rowOf(i), colOf(i));
    next[i] = nextState(grid[i] === 1, n);
  }
  grid = next;                          // swap in — never mutate in place
  generation++;
  render(); updateStats();
}

    Click & drag to toggle cells
    
        
    To seed a pattern the player paints cells on and off. On mousedown we flip the clicked cell and remember we're dragging; while dragging over other cells we set them alive too, so you can sketch shapes in one stroke. Pointer events (rather than separate mouse/touch handlers) make this work on phones as well. We only flip the array value and re-render — no simulation runs until you press Play or Step.
    
        
    
          
    let painting = false;
board.addEventListener("pointerdown", e => {
  const el = e.target.closest(".cell"); if (!el) return;
  painting = true;
  const i = +el.dataset.i;
  grid[i] = grid[i] ? 0 : 1;   // toggle the clicked cell
  render(); updateStats();
});
board.addEventListener("pointerover", e => {
  if (!painting) return;
  const el = e.target.closest(".cell"); if (!el) return;
  grid[+el.dataset.i] = 1;     // drag paints cells alive
  render(); updateStats();
});
window.addEventListener("pointerup", () => painting = false);

    7
    The play loop & adjustable speed
    
        
    Playing is just calling step() on a timer. We use setInterval whose delay is 1000 / speed milliseconds, so a speed slider of 10 means ten generations a second. Play toggles the timer on and off; changing the speed clears and restarts the interval with the new delay. Because each tick is one clean step(), pausing, single-stepping, and running fast all share the exact same logic.
    
        
    
          
    let timer = null, speed = 10;
function play(){
  if (timer) return;
  timer = setInterval(step, 1000 / speed);
}
function pause(){ clearInterval(timer); timer = null; }
speedInput.oninput = () => {
  speed = +speedInput.value;
  if (timer){ pause(); play(); }   // restart at the new rate
};

  8. 
        
    8
    Preset patterns
    
        
    Famous patterns are just lists of live-cell coordinates. We clear the grid, then stamp each [row, col] pair (offset to roughly centre it) as alive. A Glider walks diagonally forever; a Blinker flips between two states; a Pulsar is a period-3 oscillator; a Lightweight Spaceship (LWSS) glides sideways; and the Gosper Glider Gun endlessly fires new gliders. Press Play after stamping to watch them come alive.
    
        
    
          
    const PATTERNS = {
  glider:  [[0,1],[1,2],[2,0],[2,1],[2,2]],
  blinker: [[1,0],[1,1],[1,2]],
  // ...pulsar, lwss, gun: bigger coordinate lists...
};
function stamp(name, top = 2, left = 2){
  grid = new Uint8Array(CELLS);          // clear
  for (const [r, c] of PATTERNS[name])
    grid[idx(top + r, left + c)] = 1;    // set alive
  generation = 0; render(); updateStats();
}
