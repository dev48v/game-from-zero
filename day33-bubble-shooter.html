🫧 Bubble Shooter — Aim, Bounce Off the Walls, Snap to a Hex Grid, and Pop Clusters of Three

Clear the board before the bubbles pile up.

Move the mouse to aim the cannon at the bottom (or nudge with the ← → arrow keys), then click or press Space to fire the coloured bubble sitting in the barrel. The shot flies in a straight line, bounces off the side walls, and the instant it touches the ceiling or another bubble it snaps into the honeycomb grid at the nearest open cell. If that lands it against two or more of the same colour — three or more touching in total — the whole cluster pops. And here is the satisfying part: any bubbles that were only hanging on through the ones you just popped are now attached to nothing, so they drop off in a shower. A little preview shows the next bubble so you can plan two shots ahead. Bank shots off the wall to reach bubbles tucked into a corner. Clear every bubble to win; let the stack grow past the red line and it is over.

Everything here is real bubble-shooter mechanics. The board is a staggered hexagonal grid — every other row is shoved half a bubble to the side so each bubble nestles against six neighbours instead of four, which is why clusters feel round rather than blocky. The moving bubble is pure physics: an aim angle becomes a velocity, and hitting a side wall simply flips the horizontal component (vx = -vx) so the angle reflects. When it collides it snaps to the closest empty cell by rounding its pixel position back to a row and column. Popping is a breadth-first flood-fill from the cell it landed in, walking only to same-colour neighbours; if the connected group is three or more it is removed. Then a second flood-fill starts from the ceiling row and marks everything still hanging from the top — any bubble it never reaches is floating, so it drops. All vanilla JavaScript on one canvas, offline, no images or libraries.

    The staggered hex grid
    
        
    The board is rows of bubbles, but every odd row is pushed half a bubble to the right and holds one fewer bubble. That stagger is what turns a square grid into a honeycomb: each bubble ends up touching six neighbours, not four. Store it as a 2D array of colour ids (or null for empty), with the row's length depending on whether the row is even or odd.
    
        
    const R = 18;                       // bubble radius
const N = 12;                       // bubbles in an even row
const W = 2 * R * N;                // canvas width fits N of them
const rowH = R * Math.sqrt(3);      // hex rows nest tighter than 2R
const MAXROWS = 15;
// odd rows are offset half a bubble AND hold one fewer bubble
const colsInRow = r => (r % 2) ? N - 1 : N;

function newGrid(rows){
  const g = Array.from({length:MAXROWS}, (_,r) => Array(colsInRow(r)).fill(null));
  for (let r=0;r<rows;r++)
    for (let c=0;c<colsInRow(r);c++) g[r][c] = Math.random()*COLORS.length | 0;
  return g;                          // g[r][c] = colour index, or null
}

    2
    Cell ↔ pixel, and the six neighbours
    
        
    A cell's centre is its column times the diameter, plus a half-bubble shift on odd rows; the row simply steps down by rowH. The neighbour list is the whole reason to use hexes — and it depends on row parity, because the half-bubble shift changes which cells above and below actually touch.
    
        
    function cellCenter(r, c){
  const offset = (r % 2) ? R : 0;          // odd rows shifted right
  return { x: R + c*2*R + offset, y: R + r*rowH };
}
// the 6 touching cells differ for even vs odd rows
function neighbors(r, c){
  const even = (r % 2) === 0;
  const list = even
    ? [[r,c-1],[r,c+1],[r-1,c-1],[r-1,c],[r+1,c-1],[r+1,c]]
    : [[r,c-1],[r,c+1],[r-1,c],[r-1,c+1],[r+1,c],[r+1,c+1]];
  return list.filter(([nr,nc]) => nr>=0 && nr<MAXROWS && nc>=0 && nc<colsInRow(nr));
}

    Aim: an angle becomes a velocity
    
        
    The cannon sits at the bottom centre. Point at the mouse, take the angle from the cannon to the cursor, and clamp it so the shot always heads upward — never sideways or down. Firing just turns that angle into a velocity vector with a fixed speed.
    
        
    const SPEED = 8;
const ox = W/2, oy = H - 38;               // cannon origin
let aim = -Math.PI/2;                      // start pointing straight up

function aimAt(px, py){
  let a = Math.atan2(py - oy, px - ox);    // angle to the cursor
  if (a >= 0) a = (px < ox) ? -Math.PI+0.18 : -0.18;  // never fire downward
  aim = Math.max(-Math.PI+0.18, Math.min(-0.18, a));  // clamp near-horizontal
}
function fire(){
  if (state !== "aim") return;
  fly = { x:ox, y:oy, vx:Math.cos(aim)*SPEED, vy:Math.sin(aim)*SPEED, color:current };
  state = "fly";
}

    4
    Flight and wall reflection
    
        
    Advance the bubble a few sub-steps per frame so it can never tunnel through a wall or a bubble. When its centre reaches within a radius of a side wall, reflect the angle by flipping vx — that single sign flip is the whole bounce. Reaching the ceiling, or touching any placed bubble, ends the flight.
    
        
    function update(){
  if (state !== "fly") return;
  const sub = 4;                           // sub-steps stop tunnelling
  for (let i=0;i<sub;i++){
    fly.x += fly.vx/sub;  fly.y += fly.vy/sub;
    if (fly.x < R){ fly.x = R;    fly.vx = -fly.vx; }        // bounce off left
    else if (fly.x > W-R){ fly.x = W-R; fly.vx = -fly.vx; } // bounce off right
    if (fly.y <= R)  return land();          // hit the ceiling
    if (hitsGrid()) return land();           // touched a bubble
  }
}
function hitsGrid(){
  for (let r=0;r<MAXROWS;r++) for (let c=0;c<colsInRow(r);c++){
    if (grid[r][c] == null) continue;
    const p = cellCenter(r,c), dx = p.x-fly.x, dy = p.y-fly.y;
    if (dx*dx + dy*dy < (2*R)*(2*R)) return true;   // centres within a diameter
  }
  return false;
}

    5
    Snap to the nearest empty cell
    
        
    Where the bubble stopped is almost never a perfect cell centre, so round its pixel position back to a row and column, then pick the closest empty cell in that neighbourhood. That is the "snap" — it is what keeps every bubble on the honeycomb lattice no matter where it collided.
    
        
    function snapCell(x, y){
  let row = Math.round((y - R) / rowH);
  row = Math.max(0, Math.min(MAXROWS-1, row));
  let best = null, bestD = Infinity;
  for (let r=Math.max(0,row-1); r<=Math.min(MAXROWS-1,row+1); r++)
    for (let c=0;c<colsInRow(r);c++){
      if (grid[r][c] != null) continue;             // must be empty
      const p = cellCenter(r,c), d = (p.x-x)**2 + (p.y-y)**2;
      if (d < bestD){ bestD = d; best = {row:r, col:c}; }  // closest one wins
    }
  return best;
}

    6
    Flood-fill the same-colour cluster (BFS)
    
        
    From the cell the bubble just landed in, walk outward through touching neighbours, but only step onto cells of the same colour. A visited set stops loops. The list you come back with is the connected same-colour cluster — exactly the group that might pop.
    
        
    function colorCluster(r0, c0, color){
  const seen = new Set([r0+","+c0]);
  const out = [], stack = [[r0,c0]];
  while (stack.length){
    const [r,c] = stack.pop();
    if (grid[r][c] !== color) continue;
    out.push([r,c]);
    for (const [nr,nc] of neighbors(r,c)){
      const k = nr+","+nc;
      if (!seen.has(k) && grid[nr][nc] === color){ seen.add(k); stack.push([nr,nc]); }
    }
  }
  return out;                          // every same-colour bubble touching the start
}

    7
    The pop rule (three or more) and scoring
    
        
    The rule is simply a size test: if the cluster the new bubble joined has three or more bubbles, clear them all; otherwise the bubble just sticks and it becomes the next player's problem. Score the pop, and reward chains — every consecutive shot that pops something bumps a combo multiplier.
    
        
    function land(){
  const cell = snapCell(fly.x, fly.y);
  grid[cell.row][cell.col] = fly.color;              // place it
  const cluster = colorCluster(cell.row, cell.col, fly.color);
  let popped = 0, dropped = 0;
  if (cluster.length >= 3){                           // THE POP RULE
    cluster.forEach(([r,c]) => grid[r][c] = null);
    popped = cluster.length;
    dropped = dropFloating();                         // step 8
    combo++;                                          // chain bonus
  } else combo = 0;
  score += popped*10 + dropped*20 + Math.max(0,combo-1)*15;
  fly = null; nextBubble(); state = "aim";
}

    8
    Drop the floaters (a second BFS from the ceiling)
    
        
    After a pop, some bubbles may have been held up only by the ones that vanished. To find them, flood-fill from the top row through any colour: everything the fill reaches is still anchored to the ceiling. Anything it never reaches is floating — remove it. This is what makes a good shot cascade into an avalanche.
    
        
    function dropFloating(){
  const anchored = new Set(), stack = [];
  for (let c=0;c<colsInRow(0);c++)                    // seed with the ceiling row
    if (grid[0][c] != null){ anchored.add("0,"+c); stack.push([0,c]); }
  while (stack.length){                               // BFS through ANY colour
    const [r,c] = stack.pop();
    for (const [nr,nc] of neighbors(r,c)){
      const k = nr+","+nc;
      if (!anchored.has(k) && grid[nr][nc] != null){ anchored.add(k); stack.push([nr,nc]); }
    }
  }
  let dropped = 0;                                    // unreached = floating
  for (let r=0;r<MAXROWS;r++) for (let c=0;c<colsInRow(r);c++)
    if (grid[r][c] != null && !anchored.has(r+","+c)){ grid[r][c] = null; dropped++; }
  return dropped;
}

    Spawn, preview, win and lose
    
        
    Load the next bubble into the barrel and roll a fresh preview — biased to colours still on the board so the game stays winnable. Clearing every bubble wins; letting the stack sink past the danger line loses. Then it is just a render loop reading the grid onto the canvas.
    
        
    function pickColor(){
  const present = [...new Set(grid.flat().filter(v => v != null))];
  const pool = present.length ? present : COLORS.map((_,i) => i);
  return pool[Math.random()*pool.length | 0];        // stay winnable
}
