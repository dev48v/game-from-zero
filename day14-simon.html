🟢 Simon — Watch, Remember, Repeat

Watch the sequence, then play it back.

🔨 Build it yourself

    
Four pads, a growing array, a timed flash loop, live beeps from the Web Audio API, and a compare-as-you-go input check. ~90 lines of vanilla JS — copy each block in order.

    
    

      
  1. 
        
    1
    The four pads
    
        
    Simon has four coloured pads. We give each one an index — 0 green, 1 red, 2 yellow, 3 blue — and pair it with a musical note in hertz. The index is what we store and compare; the colour and tone are just how we show it. Keeping pads as plain numbers makes the sequence trivial to record and replay.
    
        
    
          
    const PADS = [
  { id: "pad-green",  freq: 392 },  // 0  G
  { id: "pad-red",    freq: 330 },  // 1  E
  { id: "pad-yellow", freq: 277 },  // 2  C#
  { id: "pad-blue",   freq: 247 },  // 3  B
];

  2. 
        
    2
    The sequence is just an array
    
        
    The whole game state is one array of pad indexes, like [2, 0, 3]. Each new round we push one more random pad onto the end, so the pattern grows by exactly one. The length of that array is the round number and, when you lose, your score. Everything else reads from this single list.
    
        
    
          
    let sequence = [];

function addStep(){
  const pad = Math.floor(Math.random() * 4);  // 0..3
  sequence.push(pad);
}
    
          
        
    
      
    3. 

      
  3. 
        
    3
    Lighting and beeping one pad
    
        
    A flash is simple: add a lit class so the pad brightens, play its tone, then remove the class a moment later. We return a Promise that resolves after the flash so the caller can wait for it. The same function is reused when you press a pad, so playback and input look and sound identical.
    
        
    
          
    function flash(pad, ms = 400){
  const el = document.getElementById(PADS[pad].id);
  el.classList.add("lit");
  beep(PADS[pad].freq, ms);
  return new Promise(res => setTimeout(() => {
    el.classList.remove("lit");
    res();
  }, ms));
}
    
          
        
    
      
    4. 

      
  4. 
        
    4
    Real beeps with the Web Audio API
    
        
    No sound files — the browser makes the tone for us. We create one AudioContext, then for each beep spin up an oscillator at the pad's frequency and a gain node to fade it out so it doesn't click. A tiny try/catch means that if audio is blocked, the game keeps running silently instead of crashing.
    
        
    
          
    let ac;
function beep(freq, ms){
  try {
    ac = ac || new (window.AudioContext || webkitAudioContext)();
    const osc = ac.createOscillator(), gain = ac.createGain();
    osc.type = "sine"; osc.frequency.value = freq;
    osc.connect(gain); gain.connect(ac.destination);
    gain.gain.setValueAtTime(0.3, ac.currentTime);
    gain.gain.exponentialRampToValueAtTime(0.0001, ac.currentTime + ms/1000);
    osc.start(); osc.stop(ac.currentTime + ms/1000);
  } catch (e) { /* audio blocked → stay silent */ }
}

    5
    Playing back the whole sequence
    
        
    To show the pattern we walk the array and await each flash, with a short gap between them. Because flash returns a Promise, await lines the flashes up one after another instead of all at once. The flash time shrinks as rounds grow, so the game speeds up. While it plays, input is locked.
    
        
    
          
    async function playback(){
  state = "showing";
  const speed = Math.max(200, 460 - sequence.length * 18);
  for (const pad of sequence){
    await flash(pad, speed);
    await wait(speed * 0.4);          // gap between flashes
  }
  state = "input"; expected = 0;      // now it's the player's turn
}
const wait = ms => new Promise(r => setTimeout(r, ms));
    
          
        
    
      
    6. 

      
  6. 
        
    6
    Capturing input and comparing
    
        
    When it's your turn, each pad click is checked immediately against sequence[expected] — the next pad we expect. A match advances the pointer; a mismatch ends the game at once. We don't wait for the whole pattern, so a wrong tap fails fast. Clicks are ignored unless the state is input.
    
        
    
          
    function press(pad){
  if (state !== "input") return;      // ignore clicks while showing
  flash(pad, 220);
  if (pad !== sequence[expected]) return gameOver();
  expected++;
  if (expected === sequence.length)   // whole pattern repeated!
    setTimeout(nextRound, 700);
}

    Next round, game over, and best score
    
        
    Clearing a round adds one step and replays the longer pattern. Losing plays a low buzzer, shows the length you reached as the score, and — if it beats your record — saves it to localStorage so your best survives a refresh. The round number is just the sequence length all the way through.
    
        
    
          
    function nextRound(){ addStep(); playback(); }

function gameOver(){
  state = "over";
  beep(110, 600);                                   // buzzer
  const score = sequence.length - 1;
  best = Math.max(best, score);
  localStorage.setItem("simonBest", best);
  showResult(score, best);
}

    Wire up the pads
    
        
    Give every pad a click handler that calls press with its index, and a Start button that resets the sequence and kicks off round one. The state variable (idle → showing → input → over) decides what each click does. Double-click the file — no server needed.
    
        
    
          
    let state = "idle", expected = 0;
let best = +localStorage.getItem("simonBest") || 0;

PADS.forEach((p, i) =>
  document.getElementById(p.id).onclick = () => press(i));

document.getElementById("start").onclick = () => {
  sequence = []; state = "showing";
  nextRound();                         // round 1
};
