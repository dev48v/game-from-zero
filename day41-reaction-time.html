⚡ Reaction Time — Wait for the Screen to Flash Green, Then Tap as Fast as You Can; Measure Your Reflex in Milliseconds Across Five Rounds With a Running Average, Your Best Time, and a Penalty for Jumping the Gun

How fast is a thought? Tap the instant it turns green.

Reaction time is the oldest lab test there is — the gap between a signal appearing and your body responding. Click the pad to start a round and it turns red with “wait for green.” After a random hidden delay of 1.5–4.5 seconds — random so you can’t learn the timing and pre-tap — the pad flashes green. The moment it does, a high-resolution stopwatch starts; tap as fast as you can and the game measures the milliseconds between the flip and your click. Do it across five rounds and it tracks every time, your best, and a running average, plus a little bar chart of all your attempts. Tap during the red phase and it’s a false start — it doesn’t count, and you retry that round, exactly like an athletics track. A typical human sits around 200–270 ms; break 250 and you’re quick. There are no images and no game library here — just a coloured div, one random timer, and performance.now().

🔨 Build it yourself

    
There is no sprite sheet and no game engine — the whole thing is one coloured div, one random timer, and a high-resolution stopwatch. The game is a five-state machine: idle until you click, wait while the pad is red, go the instant it turns green, result after your tap, and done once five rounds are in. Two ideas carry the whole thing: performance.now() gives a monotonic, sub-millisecond stopwatch so a simple subtraction is a trustworthy reaction time, and a random 1.5–4.5 s delay before green makes the flip impossible to anticipate — with a guard that turns any early tap into a no-count false start. About 90 lines of vanilla JavaScript, no libraries. Copy each block in order.

    
    

      
  1. 
        
    1
    The whole game is five states
    
        
    No objects, no scene graph — the entire game is one string that says which of five moments we are in, plus a handful of numbers. idle waits for your first click, wait is the red pause, go is the green flash, result shows a round’s time, and done is the final summary. Everything below just moves between these five states.
    
        
    const TOTAL    = 5;      // rounds in one series
const MIN_WAIT = 1500;   // shortest red delay (ms)
const MAX_WAIT = 4500;   // longest  red delay (ms)

let phase;        // "idle" | "wait" | "go" | "result" | "done"
let times = [];   // reaction time of each round (ms)
let falseStarts;  // count of too-early taps
let startTime;    // performance.now() at the green flip
let timer;        // setTimeout handle for red -> green

  2. 
        
    2
    performance.now(), not Date — a real stopwatch
    
        
    You could subtract two Date.now() readings, but you shouldn’t. Date.now() tracks wall-clock time, which the OS can nudge forward or back (NTP sync, daylight saving) and only ticks in whole milliseconds. performance.now() is a monotonic clock — it only ever counts up, from page load, in fractions of a millisecond — so the difference between two readings is a clean, trustworthy elapsed time. Perfect for a reflex measured in a few hundred ms.
    
        
    // Date.now()  → wall clock, whole ms, can jump backward  ✗
// performance.now() → monotonic, sub-ms, never jumps      ✓

const t0 = performance.now();
// ... something happens ...
const elapsedMs = performance.now() - t0;   // trustworthy

  3. 
        
    3
    Arm a round with a random hidden delay
    
        
    Starting a round paints the pad red and schedules the green flip for some point in the future. The delay is random — a uniform pick between 1.5 and 4.5 seconds — and that randomness is the whole point: if it were fixed, you would learn the rhythm and tap on a count instead of on the signal. We keep the setTimeout handle so an early tap can cancel it.
    
        
    function arm(){
  phase = "wait";
  setPad("wait", "Wait for green…",
         "Round " + (times.length + 1) + " of " + TOTAL);
  const delay = MIN_WAIT + Math.random() * (MAX_WAIT - MIN_WAIT);
  timer = setTimeout(goGreen, delay);   // flip, someday soon
}
    
      
    4. 

      
  4. 
        
    4
    Flip to green and stamp the start
    
        
    When the timer fires, the pad turns green, the state becomes go, and — in the very same instant — we read the stopwatch into startTime. This single reading is the anchor the whole measurement hangs on: it is time zero, the exact moment the signal appeared on screen. Everything the player does is measured relative to it.
    
        
    function goGreen(){
  phase = "go";
  startTime = performance.now();      // ← time zero: the flip
  setPad("go", "TAP!", "as fast as you can");
}
    
      
    5. 

      
  5. 
        
    5
    Measure the reaction — one subtraction
    
        
    Here is the measurement, and it is gloriously simple: read the stopwatch again the instant the player taps, subtract startTime, round to a whole millisecond, and that is the reaction time. Push it onto the list of round times. If five rounds are complete the series is done; otherwise show the result and wait for a click to start the next round.
    
        
    function tap(){
  const ms = Math.round(performance.now() - startTime);  // reaction!
  times.push(ms);
  if (times.length >= TOTAL){
    phase = "done"; finish();
  } else {
    phase = "result";
    setPad("result", ms + " ms", "click for round " + (times.length + 1));
  }
  updateHUD();
}
    
      
    6. 

      
  6. 
        
    6
    Punish the early tap
    
        
    Without a guard you could just hammer the pad and “win.” So a tap during the red wait phase is a false start: cancel the pending green flip so it can’t fire late, count the false start, and drop back to idle so the player retries the same round — it never counts as a time. This is exactly the jump-the-gun rule from an athletics track.
    
        
    function tooSoon(){
  clearTimeout(timer);         // kill the pending green flip
  falseStarts++;
  phase = "idle";              // retry this round — nothing recorded
  setPad("tooearly", "Too soon!",
         "you tapped before green — click to retry this round");
  updateHUD();
}

    7
    Best and running average
    
        
    Because every round time lives in one array, the summary stats are one-liners. The best is Math.min over the list; the average is the sum divided by the count. A small rating turns a number into words — under 250 ms is quick, 200–270 ms is the normal human range. The final screen just reports the average and the best.
    
        
    function finish(){
  const best = Math.min(...times);
  const avg  = Math.round(times.reduce((a,b) => a + b, 0) / times.length);
  setPad("done", avg + " ms",
         "average · best " + best + " ms · click to play again");
}
function rate(ms){                     // number → words
  if (ms < 200) return "Superhuman";
  if (ms < 250) return "Lightning";
  if (ms < 300) return "Sharp";
  if (ms < 400) return "Average";
  return "Sluggish";
}
    
      
    8. 

      
  8. 
        
    8
    One click handler routes every state
    
        
    The same tap means something different in each phase, so one router reads phase and dispatches. Idle → arm a round. Wait (red) → too soon. Go (green) → measure. Result → arm the next round. Done → reset for a fresh series. That five-way switch is the game loop; there is nothing else driving it.
    
        
    function handleTap(){
  if      (phase === "idle")   arm();      // start a round
  else if (phase === "wait")   tooSoon();  // red → false start
  else if (phase === "go")     tap();      // green → measure
  else if (phase === "result") arm();      // next round
  else if (phase === "done")   reset();    // new series
}

  9. 
        
    9
    The click zone and keyboard
    
        
    Wiring is tiny. The big coloured pad listens for a click and calls the router; setPad swaps a CSS class to change its colour and writes the two lines of text. For serious speed the keyboard beats the mouse, so Space and Enter also tap (we call preventDefault so the page doesn’t scroll), and R restarts.
    
        
    function setPad(state, big, sub){
  pad.className = "pad " + state;     // .idle/.wait/.go/.result/.done → colour
  padBig.textContent = big;
  padSub.textContent = sub;
}
pad.addEventListener("click", handleTap);
document.addEventListener("keydown", e => {
  if (e.code === "Space" || e.key === "Enter"){ e.preventDefault(); handleTap(); }
  else if (e.key.toLowerCase() === "r") reset();
});

    Draw the histogram of attempts
    
        
    A column of numbers is dull; five bars tell the story at a glance. Map each time onto a fixed 600 ms scale to get a bar width, colour it by how fast it was — green for a flyer, red for a slow one — and lay the five rows out. Empty rounds show a stub. That is the whole “chart”: no library, just divs with a width and a background colour.
    
        
    function speedColor(ms){                 // fast = green, slow = red
  if (ms < 250) return "#22c55e";
  if (ms < 350) return "#84cc16";
  if (ms < 450) return "#f59e0b";
  return "#ef4444";
}
function renderHisto(){
  histoEl.innerHTML = "";
  for (let i = 0; i < TOTAL; i++){
    const ms  = times[i];                              // may be undefined
    const pct = ms != null ? Math.min(100, ms / 600 * 100) : 0;
    histoEl.insertAdjacentHTML("beforeend",
      '<div class="hbar-row"><span class="hbar-lbl">R' + (i+1) + '</span>' +
      '<span class="hbar-track"><span class="hbar-fill" style="width:' + pct +
      '%;background:' + (ms!=null?speedColor(ms):'transparent') + '"></span></span>' +
      '<span class="hbar-val">' + (ms!=null?ms+' ms':'—') + '</span></div>');
  }
}
