🔴 Checkers — Jump to Capture, Chain Your Multi-Jumps, and King Your Men on the Back Rank

Play Checkers against the AI.

You are red, the AI is black, and every piece lives on the dark squares only. A plain man moves one square diagonally forward to an empty dark square — your men move up, the AI's move down. You capture by jumping: hop diagonally over a touching enemy piece into the empty square just beyond, and that piece is removed. Captures are mandatory — if any jump is available you must take one — and after a jump, if the same piece can jump again it must keep going, so a single turn can sweep several pieces off the board. Reach the far back rank and your man is crowned a king, free to move and jump in both diagonal directions. You win when the AI has no pieces left, or no legal move. The emerald outlines mark the pieces you may move; click one, then click a green square. The AI plays a real minimax search with alpha-beta pruning — pick how many moves ahead it looks.

You: 12
AI: 12
👑 Kings: You 0 · AI 0
Turn: You

Everything here is real Checkers (English draughts). The board is one 8×8 array of small integers — 0 empty, and separate codes for your men and kings and the AI's men and kings — with pieces only ever on the dark squares. One move generator produces every legal move: it first hunts for jumps by recursing over the multi-jump tree (hop an enemy, land beyond, then try to jump again from there), and only if no jump exists anywhere does it fall back to plain diagonal slides — that is the mandatory-capture rule in code. Landing on the back rank crowns the piece, which flips its allowed directions. The AI runs the same minimax you would write by hand: it plays each of its moves, imagines your best reply, and so on down to the chosen depth, scoring leaf positions by material (kings count far more than men) plus a little positional sense — hold the centre, push men forward, guard the back row — and alpha-beta pruning throws away branches that cannot change the decision. All vanilla JavaScript on a canvas, offline, no images or libraries.

    The state: 12 men each, on the dark squares
    
        
    The whole game is one 8×8 grid of integers: 0 empty, then a code each for your man/king and the AI's man/king. Pieces only ever sit on the dark squares — where (row+col) is odd — so three rows of them at each end give twelve pieces a side. You move first.
    
        
    const EMPTY=0, HM=1, HK=2, AM=3, AK=4;      // empty, human man/king, AI man/king
const HUMAN=1, AI=2;
const inB    = (r,c) => r>=0 && r<8 && c>=0 && c<8;
const isDark = (r,c) => ((r+c) & 1) === 1;   // pieces live on dark squares only
function newBoard(){
  const b = Array.from({length:8}, () => Array(8).fill(EMPTY));
  for (let r=0;r<3;r++) for (let c=0;c<8;c++) if (isDark(r,c)) b[r][c]=AM; // AI: top 3 rows
  for (let r=5;r<8;r++) for (let c=0;c<8;c++) if (isDark(r,c)) b[r][c]=HM; // you: bottom 3
  return b;                                  // 12 men each — you move first
}

    2
    Men move one way, kings move both
    
        
    A few tiny helpers decode a square. The key one is stepDirs: a king slides along all four diagonals, but a plain man only goes forward — your men up the board, the AI's men down. Every rule below is just a loop over these direction steps.
    
        
    const owner  = p => p===HM||p===HK ? HUMAN : p===AM||p===AK ? AI : 0;
const isKing = p => p===HK || p===AK;
const king   = p => p===HM ? HK : p===AM ? AK : p;   // crown a man
const opp    = pl => pl===HUMAN ? AI : HUMAN;

// men move one way; kings move both. Your men go UP (row-1), AI men go DOWN.
function stepDirs(p){
  if (isKing(p)) return [[-1,-1],[-1,1],[1,-1],[1,1]];
  return owner(p)===HUMAN ? [[-1,-1],[-1,1]] : [[1,-1],[1,1]];
}
const reachesBack = (r,pl) => pl===HUMAN ? r===0 : r===7;  // far side = promotion

  3. 
        
    3
    Plain slides: one step to an empty square
    
        
    The non-capturing move is the simplest: for each allowed direction, look at the adjacent dark square; if it is empty, that is a legal slide. We describe every move the same way — a from, a to, and a (here empty) list of captured squares — so the rest of the engine never cares which kind it is.
    
        
    // a plain slide: one diagonal step to an adjacent EMPTY square
function simpleMoves(b, r, c){
  const p = b[r][c], out = [];
  for (const [dr,dc] of stepDirs(p)){
    const nr = r+dr, nc = c+dc;
    if (inB(nr,nc) && b[nr][nc]===EMPTY)
      out.push({ from:[r,c], to:[nr,nc], captures:[], steps:[[nr,nc]] });
  }
  return out;
}

  4. 
        
    4
    Jumps, and the multi-jump recursion
    
        
    A capture hops an adjacent enemy into the empty square two diagonals away. The twist is chaining: after landing, the same piece must try to jump again. So we simulate the hop on a copied board and recurse from the landing square, threading the list of captured squares so we never jump the same piece twice. Crowning ends the turn, so we stop recursing there.
    
        
    // every jump chain from (r,c): hop an adjacent enemy into the empty square
// beyond, then recurse — the same piece keeps jumping while it can.
function jumpsFrom(b, r, c, p, captured){
  const res = [];
  for (const [dr,dc] of stepDirs(p)){
    const mr=r+dr, mc=c+dc, lr=r+2*dr, lc=c+2*dc;      // mid = jumped, land = target
    if (!inB(lr,lc) || b[lr][lc]!==EMPTY) continue;     // must land on empty
    const mid = b[mr][mc];
    if (mid===EMPTY || owner(mid)===owner(p)) continue; // must hop an ENEMY
    if (captured.some(q => q[0]===mr && q[1]===mc)) continue; // never twice
    const nb = b.map(row => row.slice());               // simulate the hop
    nb[r][c]=EMPTY; nb[mr][mc]=EMPTY;
    let np=p, crowned=false;
    if (!isKing(p) && reachesBack(lr, owner(p))){ np=king(p); crowned=true; }
    nb[lr][lc]=np;
    const cap  = captured.concat([[mr,mc]]);
    const cont = crowned ? [] : jumpsFrom(nb, lr, lc, np, cap); // crowning ends turn
    if (!cont.length) res.push({ to:[lr,lc], captures:cap, steps:[[lr,lc]] });
    else for (const k of cont)
      res.push({ to:k.to, captures:k.captures, steps:[[lr,lc]].concat(k.steps) });
  }
  return res;
}

  5. 
        
    5
    Move generation — capture is mandatory
    
        
    Here is the rule that shapes the whole game: if any jump exists for the side to move, only jumps are legal — slides are forbidden. So gather every jump first; if the list is non-empty, return it. Only when nobody can jump do we fall back to the plain slides. This one if is the mandatory-capture rule.
    
        
    // MANDATORY capture: if ANY jump exists for the side, only jumps are legal.
function genMoves(b, pl){
  const jumps = [];
  for (let r=0;r<8;r++) for (let c=0;c<8;c++)
    if (owner(b[r][c])===pl)
      for (const j of jumpsFrom(b, r, c, b[r][c], []))
        jumps.push({ from:[r,c], to:j.to, captures:j.captures, steps:j.steps });
  if (jumps.length) return jumps;              // forced to capture
  const slides = [];
  for (let r=0;r<8;r++) for (let c=0;c<8;c++)
    if (owner(b[r][c])===pl) slides.push(...simpleMoves(b, r, c));
  return slides;
}

    6
    Apply a move: remove captures, crown kings
    
        
    Applying a move lifts the piece off its start square, erases every captured square the jump chain collected, and drops the piece on its final square — promoting it to a king if that square is the back rank. Work on a copy so the AI can explore hypothetical moves without disturbing the real board.
    
        
    function applyMove(b, mv, pl){
  const nb = b.map(row => row.slice());          // copy — safe for AI search
  let p = nb[mv.from[0]][mv.from[1]];
  nb[mv.from[0]][mv.from[1]] = EMPTY;             // lift the piece
  for (const [cr,cc] of mv.captures) nb[cr][cc] = EMPTY;  // remove every jumped man
  const [fr,fc] = mv.to;
  if (!isKing(p) && reachesBack(fr, pl)) p = king(p);     // crown on the back rank
  nb[fr][fc] = p;
  return nb;
}
function counts(b){
  let you=0, ai=0;
  for (let r=0;r<8;r++) for (let c=0;c<8;c++){
    const o = owner(b[r][c]); if (o===HUMAN) you++; else if (o===AI) ai++;
  }
  return { you, ai };
}

    Scoring a position: material + a little geometry
    
        
    To judge a leaf position we sum piece values from the AI's point of view: a man is 100, a king is worth about 1.75 men. Then a dash of positional sense — a bonus for controlling the centre files, for men that have advanced toward promotion, and for men still guarding their own back row (which blocks the opponent from crowning).
    
        
    const MAN = 100, KING = 175;                     // a king is worth ~1.75 men
function evaluate(b){                            // + = good for the AI
  let s = 0;
  for (let r=0;r<8;r++) for (let c=0;c<8;c++){
    const p = b[r][c]; if (!p) continue;
    const mine = owner(p)===AI;
    let v = isKing(p) ? KING : MAN;
    if (c>=2 && c<=5) v += 4;                      // centre files are stronger
    if (!isKing(p)){
      v += (owner(p)===AI ? r : 7-r) * 3;          // reward men that advance
      if ((owner(p)===AI && r===0) ||
          (owner(p)===HUMAN && r===7)) v += 6;      // guard your back row
    }
    s += mine ? v : -v;
  }
  return s;
}

    Minimax with alpha-beta pruning
    
        
    Minimax looks ahead: the AI picks the move that maximises the score, assuming you reply to minimise it, alternating down to depth plies. A position with no legal move is terminal — that side has lost — scored a huge value nudged by depth so the AI prefers to win sooner and lose later. Alpha-beta carries the best-so-far bounds and prunes branches that cannot change the choice.
    
        
    function minimax(b, depth, alpha, beta, toMove){
  const moves = genMoves(b, toMove);
  if (!moves.length)                              // no move = that side LOSES
    return toMove===AI ? -1e6 - depth : 1e6 + depth;  // prefer quicker wins
  if (depth === 0) return evaluate(b);
  if (toMove === AI){                             // maximising player
    let best = -Infinity;
    for (const m of moves){
      best = Math.max(best, minimax(applyMove(b,m,AI), depth-1, alpha, beta, HUMAN));
      alpha = Math.max(alpha, best);
      if (beta <= alpha) break;                    // prune
    }
    return best;
  } else {                                        // minimising player
    let best = Infinity;
    for (const m of moves){
      best = Math.min(best, minimax(applyMove(b,m,HUMAN), depth-1, alpha, beta, AI));
      beta = Math.min(beta, best);
      if (beta <= alpha) break;                    // prune
    }
    return best;
  }
}

    Pick the move, then wire up the loop
    
        
    bestMove tries each AI move and keeps the one with the highest minimax value — always a legal move, since it only ever iterates real moves. The loop is then simple: a side with no move has lost; otherwise the human clicks a piece and a destination, and the AI answers by searching. Rendering is a pure function of the array.
    
        
    function bestMove(b, depth){
  let best=null, bestV=-Infinity;
  for (const m of genMoves(b, AI)){
    const v = minimax(applyMove(b,m,AI), depth-1, -Infinity, Infinity, HUMAN);
    if (v > bestV){ bestV = v; best = m; }
  }
  return best;                                    // always a legal AI move
}
