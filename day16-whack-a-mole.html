🔨 Whac-a-Mole — Fast Hands Win

Moles keep popping up — whack as many as you can before the clock runs out.

Nine holes, one timer, and a steady stream of moles. Each mole pops up in a random empty hole and only stays up for a short moment before ducking back down. Click or tap a mole while it's up to bonk it: that scores a point and builds a combo. The longer you survive, the faster moles appear and the quicker they hide, so your reactions have to keep up. You have 30 seconds. Whack fast, whack often, and beat your best.

A combo grows with every mole you hit in a row; let one escape un-whacked and the combo resets to zero. 🔨 Build it yourself

    
A 3×3 grid of holes, an array that tracks which hole is "up", a spawn loop on a timer, a short up-window per mole, a click handler that scores hits, a countdown, and a difficulty ramp that tightens the timing as the clock drops. ~90 lines of vanilla JS — copy each block in order.

    
    

      
  1. 
        
    1
    Build the 3×3 grid of holes
    
        
    There's no canvas here — the board is nine plain <div> holes laid out by CSS grid. We create them in a loop, drop a hidden mole element inside each, and remember every hole in an array so we can reach any of them by index later. The array is the whole game's memory: hole 0 to hole 8, each one either empty or holding a popped-up mole.
    
        
    
          
    const board = document.getElementById("board");
const holes = [];                        // index 0..8
for (let i = 0; i < 9; i++){
  const hole = document.createElement("div");
  hole.className = "hole";
  hole.innerHTML = '<div class="mole">🐹</div>';
  hole.dataset.i = i;                    // remember its slot
  board.appendChild(hole);
  holes.push(hole);
}
    
          
        
    
      
    2. 

      
  2. 
        
    2
    Pick a random empty hole
    
        
    A mole should never pop up where one is already showing, or it would look like nothing happened. So before spawning we build a list of holes that aren't currently up, then pick one at random from that list. Math.random() gives a number from 0 up to (but not including) 1; multiply by the list length and floor it to land on a valid index. If every hole is busy, we simply skip this tick.
    
        
    
          
    function freeHole(){
  const free = holes.filter(h => !h.classList.contains("up"));
  if (free.length === 0) return null;          // all busy → skip
  return free[Math.floor(Math.random() * free.length)];
}
    
          
        
    
      
    3. 

      
  3. 
        
    3
    Pop a mole up, then auto-hide it
    
        
    To show a mole we add the up class — CSS slides the 🐹 into view. Then we set a single setTimeout for the "up window": when it fires, if that mole is still up (you didn't whack it) it ducks down again and the combo breaks. We store the timeout id on the hole so a successful whack can cancel it. That stored handle is the key to clean hit detection in the next step.
    
        
    
          
    function popUp(hole, upTime){
  hole.classList.add("up");
  hole._timer = setTimeout(() => {
    if (hole.classList.contains("up")){       // escaped un-whacked
      hole.classList.remove("up");
      combo = 0;                              // missing breaks the combo
      comboEl.textContent = combo;
    }
  }, upTime);
}
    
          
        
    
      
    4. 

      
  4. 
        
    4
    Click to whack — score and combo
    
        
    One click listener on each hole handles the whack. A click only counts if the hole is actually up and we haven't already scored it (a guard stops a frantic double-tap from counting twice). On a valid hit we cancel the auto-hide timer, drop the mole, bump the combo, and add points — bigger combos are worth more. A quick bonk animation gives the satisfying squash before it disappears.
    
        
    
          
    holes.forEach(hole => hole.addEventListener("pointerdown", () => {
  if (state !== "play") return;
  if (!hole.classList.contains("up") || hole._hit) return;  // guard
  hole._hit = true;
  clearTimeout(hole._timer);              // cancel the escape
  combo++;
  score += 1 + Math.floor(combo / 3);     // combos pay more
  scoreEl.textContent = score;
  comboEl.textContent = combo;
  hole.classList.add("bonk");
  setTimeout(() => { hole.classList.remove("up","bonk"); hole._hit = false; }, 120);
}));
    
          
        
    
      
    5. 

      
  5. 
        
    5
    The spawn loop on a timer
    
        
    Instead of a fixed-interval timer, each spawn schedules the next one with setTimeout. That lets the delay change every time, which is what makes the ramp possible. Each tick we grab a free hole, compute how long the mole should stay up, pop it, then book the next spawn after a random-ish delay. We keep the timeout id in spawnTimer so game-over can stop the chain cleanly.
    
        
    
          
    let spawnTimer = null;
function spawn(){
  const hole = freeHole();
  if (hole) popUp(hole, upWindow());        // upWindow shrinks over time
  spawnTimer = setTimeout(spawn, spawnDelay());   // schedule the next
}
    
          
        
    
      
    6. 

      
  6. 
        
    6
    The 30-second countdown
    
        
    A separate setInterval ticks once per second, subtracting one from timeLeft and updating the display. The countdown is deliberately independent of the spawn loop — one governs how long the round lasts, the other governs how often moles appear. When the clock hits zero we stop the interval and call game-over. Two simple timers, two clear jobs.
    
        
    
          
    let timeLeft = 30, clock = null;
function startClock(){
  clock = setInterval(() => {
    timeLeft--;
    timeEl.textContent = timeLeft;
    if (timeLeft <= 0) gameOver();
  }, 1000);
}
    
          
        
    
      
    7. 

      
  7. 
        
    7
    Difficulty that ramps with the clock
    
        
    No level numbers — the game just gets harder as time passes. We measure elapsed time as a fraction from 0 (start) to 1 (end), then use it to shrink both the gap between spawns and how long each mole stays up. Early on moles are slow and generous; near the end they flicker by. A little randomness on top keeps the pattern from feeling robotic.
    
        
    
          
    function progress(){ return (30 - timeLeft) / 30; }   // 0 → 1

function spawnDelay(){
  const t = progress();
  return 900 - t * 550 + Math.random() * 250;   // ~900ms → ~350ms
}
function upWindow(){
  const t = progress();
  return 1100 - t * 650;                         // ~1100ms → ~450ms up
}
    
          
        
    
      
    8. 

      
  8. 
        
    8
    Game over, best score, and restart
    
        
    When time runs out we flip state to over, stop both timers, and pull down any mole still up. We compare the run to the best score in localStorage and save it if you beat it, so your record survives a refresh. Start/Restart resets the score, combo and clock, clears every hole, sets state to play, and kicks off the spawn loop and countdown again — a clean round every time. Double-click the file; no server needed.
    
        
    
          
    function gameOver(){
  state = "over";
  clearInterval(clock); clearTimeout(spawnTimer);
  holes.forEach(h => h.classList.remove("up","bonk"));
  if (score > best){ best = score; localStorage.setItem("wamBest", best); }
  bestEl.textContent = best;
  showOverlay("Time!", "Score " + score + " · best " + best);
}
function start(){
  state = "play"; score = 0; combo = 0; timeLeft = 30;
  scoreEl.textContent = comboEl.textContent = 0; timeEl.textContent = 30;
  holes.forEach(h => { h.classList.remove("up","bonk"); h._hit = false; });
  overlay.classList.add("hidden");
  startClock(); spawn();
}
