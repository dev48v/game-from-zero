🦘 Doodle Jump — Bounce Your Way Up

Bounce higher and higher — don't fall off the bottom.

Your character bounces on its own: every time it lands on a platform it springs straight back up. Your only job is steering left and right to line up the next platform. Use ←/→ or A/D (or tap/click the left and right half of the screen). Run off one edge and you reappear on the other. As you climb, the view scrolls up and fresh platforms keep appearing above. Miss them all and drop off the bottom — that's game over. Your score is the highest you reached.

🦘
Doodle Jump
Steer with ←/→ or A/D. The bounce is automatic.

The jump is constant — you never tap to jump, you only aim. Platform spacing is slightly random and widens as you go, so the climb gets harder the higher you get. Everything runs offline in one canvas with requestAnimationFrame; on touch screens, hold the left or right half of the canvas to steer.

🔨 Build it yourself

    
A canvas, a falling player pulled by gravity, an array of platforms, a bounce on landing, a camera that scrolls the world down, and a spawn-and-cull loop that keeps the climb endless. ~90 lines of vanilla JS — copy each block in order.

    
    

      
  1. 
        
    1
    Canvas and the game loop
    
        
    Everything is drawn on one <canvas>. The loop runs through requestAnimationFrame, which calls our function ~60 times a second in sync with the screen. Each frame we do three things in order: update the physics, draw the new picture, then ask for the next frame. That update-then-draw rhythm is the heartbeat of every action game.
    
        
    
          
    const cv = document.getElementById("game");
const ctx = cv.getContext("2d");
const W = cv.width, H = cv.height;

function loop(){
  update();              // move things
  draw();                // paint the frame
  requestAnimationFrame(loop);
}
requestAnimationFrame(loop);

    2
    The player, gravity, and the auto-bounce
    
        
    The player has a position and a vertical velocity vy. Every frame gravity adds a little to vy so the fall speeds up, and vy is added to y so the player actually moves. There is no jump button: whenever the player lands on a platform we slam vy to a fixed negative number, launching it straight back up. Constant impulse in, gravity pulling back — that loop is the bounce.
    
        
    
          
    const player = { x: W/2, y: H-80, w: 34, h: 34, vx: 0, vy: 0 };
const GRAVITY = 0.45;
const JUMP = -12;          // constant upward impulse

function applyGravity(){
  player.vy += GRAVITY;    // fall faster every frame
  player.y  += player.vy;  // velocity moves the player
}
    
          
        
    
      
    3. 

      
  3. 
        
    3
    Horizontal movement with edge wrap
    
        
    Left/right keys set the horizontal velocity vx, which we add to x each frame. Instead of stopping at the walls, the player wraps: walk off the right edge and you reappear on the left, and the other way round. Wrapping keeps the whole width usable and lets you slip across the screen to catch a platform on the far side.
    
        
    
          
    function moveSideways(){
  player.x += player.vx;
  if (player.x > W) player.x = 0;          // off right → left
  if (player.x < 0) player.x = W;          // off left  → right
}
// keys: ArrowLeft / A → vx = -5 · ArrowRight / D → vx = 5
    
          
        
    
      
    4. 

      
  4. 
        
    4
    Platforms, and bouncing only while falling
    
        
    Platforms are just objects with an x, a y, and a width. We only check for a landing when the player is moving downward (vy > 0) and its feet cross the top of a platform. That one rule lets you pass straight up through platforms and only catch them on the way down — exactly how Doodle Jump feels. On a hit, we set vy to the jump impulse and the bounce begins again.
    
        
    
          
    function checkLanding(){
  if (player.vy <= 0) return;              // only when FALLING
  for (const p of platforms){
    const feet = player.y + player.h;
    if (player.x + player.w > p.x && player.x < p.x + p.w &&
        feet > p.y && feet < p.y + 14){
      player.vy = JUMP;                    // bounce!
    }
  }
}

    5
    The scrolling camera — move the world, not the player
    
        
    The player never really climbs the screen — it stays near the middle while the world slides down past it. When the player rises above a line, we measure how far and push everything else — every platform — down by that amount, while also adding it to the score. From the player's point of view it's climbing; from the code's point of view, the whole world is scrolling beneath a fixed camera.
    
        
    
          
    function scroll(){
  const line = H * 0.4;                    // camera follow line
  if (player.y < line){
    const dy = line - player.y;
    player.y = line;                       // pin the player
    for (const p of platforms) p.y += dy;  // push world DOWN
    score += Math.round(dy);               // height climbed
  }
}
    
          
        
    
      
    6. 

      
  6. 
        
    6
    Spawning above and culling below
    
        
    As platforms scroll down, the ones near the top thin out and the ones past the bottom are useless. So each frame we drop any platform that has fallen below the screen, and while the highest platform isn't high enough we add a new one just above it. Spacing is slightly randomized and grows as the score climbs, which is what quietly ramps up the difficulty.
    
        
    
          
    function recycle(){
  platforms = platforms.filter(p => p.y < H);     // cull off-screen
  while (topY() > 0){                              // fill the gap above
    const gap = 70 + Math.random()*40 + score*0.002;
    platforms.push({ x: Math.random()*(W-64), y: topY()-gap, w: 64 });
  }
}
const topY = () => Math.min(...platforms.map(p => p.y));

    7
    Game over and restart
    
        
    There is exactly one way to lose: fall past the bottom of the screen. When the player's top edge drops below the canvas height, we flip the state to over, save the best score to localStorage, and show the overlay. Restart resets the player, rebuilds the starting platforms, zeroes the score, and flips the state back to play — a clean slate every time.
    
        
    
          
    function update(){
  if (state !== "play") return;
  applyGravity(); moveSideways(); checkLanding(); scroll(); recycle();
  if (player.y > H){                       // fell off the bottom
    state = "over";
    best = Math.max(best, score);
    localStorage.setItem("djBest", best);
    showOverlay("Game Over", "Score " + score);
  }
}

    Wire up controls and start
    
        
    Key-down sets vx, key-up clears it, so the player drifts only while a key is held. We also let a tap on the left or right half of the canvas steer, for touch screens. The Start button calls reset() to lay down the first platforms under the player and set the state to play. Double-click the file — no server needed.
    
        
    
          
    addEventListener("keydown", e => {
  if (e.key === "ArrowLeft"  || e.key === "a") player.vx = -5;
  if (e.key === "ArrowRight" || e.key === "d") player.vx =  5;
});
addEventListener("keyup", () => player.vx = 0);

cv.addEventListener("pointerdown", e => {
  const x = e.offsetX;                     // half-screen touch steer
  player.vx = x < W/2 ? -5 : 5;
});
cv.addEventListener("pointerup", () => player.vx = 0);

document.getElementById("startBtn").onclick = reset;
