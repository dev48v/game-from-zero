🀄 Mahjong Solitaire — Clear the Layered Board by Matching Free Pairs of Identical Tiles

Take the whole stack apart, two tiles at a time.

Tiles are stacked into a little layered pyramid, and most of them are stuck — pinned under a tile above, or wedged between neighbours on both sides. A tile is free only when nothing sits on top of it and at least one of its left or right sides is open. Your only move is to click two free tiles that show the same face; they vanish as a pair, and whatever they were covering or blocking becomes reachable. Peel the board apart until every tile is gone. Choose badly and you can strand yourself with no matching free pair left — that is a deadlock, and the board tells you the moment it happens so you can hit 🔀 Shuffle and get a fresh, still-solvable arrangement. There is no luck to the deal: every board is generated so a full solution exists.

Every tile in this game is nothing more than three numbers — a column, a row, and a layer — plus the face it is showing. The layered look you see is drawn from those numbers: higher layers are nudged up and to the left so the ones underneath peek out. Two rules run the whole game. First, the free test: a tile is playable only if no other tile overlaps its footprint from a higher layer, and either its left or its right side has no neighbour on the same layer. Second, the match test: two free tiles clear only if their faces are identical. The deal itself is built backwards — the code repeatedly removes free pairs from a full board and stamps each pair a face as it goes, so replaying that order is always a guaranteed win. After every move a quick scan asks whether any matchable free pair still exists; if none does, the board is deadlocked and offers a reshuffle that lays the remaining faces back down into a fresh, solvable pattern. All vanilla JavaScript on one canvas, offline, no images and no libraries.

🔨 Build it yourself

    
A tile is three numbers and a face. One overlap test answers both "is anything on top of me?" and "who is beside me?". The free rule is that test spelled out; the match rule is one string comparison. The clever part is the deal — build it by tearing a full board apart into free pairs and stamping faces as you go, and replaying that order is a guaranteed win. Add a deadlock scan, a reshuffle that stays solvable, and an isometric draw that offsets each layer up and to the left. About 150 lines of vanilla JavaScript on one canvas — no images, no libraries.     A tile is three numbers and a face
    
        
    There is no sprite and no grid array — the whole board is just a flat list of little objects. Each tile knows its x column and y row (measured in half-tile units so layers can straddle each other), its z layer, the face it shows, and whether it has been cleared. Everything the game does is a question asked of this list.
    
        
    const FACES = ["🀄","🎋","🌸","🔥","💧","⭐","🌙","🍀","🔔"];  // 9 faces × 4 copies = 36 tiles

function buildTiles(){
  tiles = []; let id = 0;
  const push = (x, y, z) => tiles.push({ x, y, z, id: id++, face: null, gone: false });
  for (let r = 0; r < 3; r++) for (let c = 0; c < 8; c++) push(c*2, r*2, 0);   // base: 8×3
  for (let r = 0; r < 2; r++) for (let c = 0; c < 6; c++) push(c*2+2, r*2+1, 1); // top: 6×2, offset
}

    One overlap test does everything
    
        
    Each tile covers a 2×2 square in those half-units. Because the unit is half a tile, two tiles overlap exactly when their columns differ by less than 2 and their rows differ by less than 2. This single line is reused for "is a tile sitting on top of me?" and, restricted to one axis, for "is a tile directly beside me?". Get this right and the rest is easy.
    
        
    // footprints overlap when they are closer than a full tile on BOTH axes
function overlapXY(a, b){
  return Math.abs(a.x - b.x) < 2 && Math.abs(a.y - b.y) < 2;
}

    3
    The free rule: uncovered AND a side open
    
        
    This is the heart of Mahjong Solitaire. A tile is free — and therefore clickable — only when two things are both true. Nothing may overlap it from a higher layer (nothing on top), and it must have no neighbour on at least one side, left or right, at its own layer. A tile pinned under another, or walled in on both sides, is locked until you remove what is trapping it.
    
        
    function isFree(t, gone = u => u.gone){
  if (gone(t)) return false;
  // 1) nothing on top: no live tile on a higher layer overlaps my footprint
  for (const u of tiles)
    if (u !== t && !gone(u) && u.z > t.z && overlapXY(u, t)) return false;
  // 2) a side is open: check for same-layer neighbours immediately left / right
  let left = false, right = false;
  for (const u of tiles){
    if (u === t || gone(u) || u.z !== t.z) continue;
    if (Math.abs(u.y - t.y) < 2){
      if (u.x === t.x - 2) left  = true;
      if (u.x === t.x + 2) right = true;
    }
  }
  return !left || !right;                     // free if either side is clear
}

    The match rule: same face, both free
    
        
    Clearing a pair is deliberately simple. Click one free tile to select it, click a second free tile, and if their faces are identical both disappear. Clicking the selected tile again cancels; clicking a different free tile just moves your selection. Every removal can only free more tiles, never trap them, which is why the board opens up as you play.
    
        
    if (selected && selected.face === t.face){   // two free tiles, identical face
  selected.gone = t.gone = true;             // clear the pair
  undoStack.push([selected, t]);
  matches++; selected = null;
} else {
  selected = t;                              // pick / switch selection
}

    5
    Deal a board that can actually be won
    
        
    Sprinkling faces at random makes boards that are often impossible. The trick is to build the deal backwards: pretend the board is full, repeatedly find the currently free tiles, remove two of them, and stamp that pair a face. Because every pair you stamp was free at the moment you removed it, replaying the removals in order is a guaranteed solution. Faces are drawn from a bag so each appears exactly four times.
    
        
    function generateSolvable(activeIds, faceCounts){
  const present = {}; activeIds.forEach(id => present[id] = true);
  const gone = u => !present[u.id];
  const bag = [];                                    // one entry per PAIR
  for (const f in faceCounts) for (let i = 0; i < faceCounts[f] / 2; i++) bag.push(f);
  shuffle(bag);
  const assign = {}, ids = activeIds.slice();
  let left = ids.length;
  while (left > 0){
    const free = ids.filter(id => present[id] && isFree(tiles[id], gone));
    if (free.length < 2) return null;                // stuck → caller retries
    shuffle(free);
    const a = free[0], b = free[1], f = bag.pop();
    assign[a] = assign[b] = f;                        // this pair matches
    present[a] = present[b] = false; left -= 2;       // ...remove and continue
  }
  return assign;                                      // a provably solvable deal
}

    Detecting a deadlock after every move
    
        
    A guaranteed-solvable deal can still be played into a dead end, because with four copies of each face you can pair the wrong two. So after every match we ask one question: among the tiles that are free right now, does any face appear at least twice? Group the free tiles by face; the first face that reaches a count of two means a move exists. No such face means deadlock.
    
        
    function freeTiles(){ return tiles.filter(t => !t.gone && isFree(t)); }

function hasMove(){
  const count = {};
  for (const t of freeTiles()){
    count[t.face] = (count[t.face] || 0) + 1;
    if (count[t.face] >= 2) return true;      // two free tiles share a face
  }
  return false;                               // nothing matchable → deadlock
}

    Rescue: reshuffle, still solvable
    
        
    When you deadlock — or just want a way out — the shuffle does not scatter faces blindly. It takes the faces still on the board (each remaining count is always even, because you clear in pairs) and runs the very same backwards generator over just the tiles that are left. The result is a brand-new layout of those faces that is, once again, guaranteed to have a full solution.
    
        
    function shuffleBoard(){
  const active = tiles.filter(t => !t.gone).map(t => t.id);
  const counts = {};
  for (const id of active) counts[tiles[id].face] = (counts[tiles[id].face] || 0) + 1;
  let deal = null, tries = 0;
  while (!deal && tries++ < 300) deal = generateSolvable(active, counts);  // stay solvable
  for (const id of active) tiles[id].face = deal[id];
  selected = null; draw();
}

    Draw the stack in fake 3D
    
        
    Half-unit columns and rows become pixels by scaling, and each higher layer is nudged up and to the left by a fixed amount so the tiles below peek out from under it. Draw the live tiles back-to-front — lowest layer first, then top to bottom — and the overlaps naturally look like a raised stack. Each tile is an ivory rounded rectangle with a darker edge and a big emoji face.
    
        
    const TW = 46, TH = 60, EDGE = 6, LDX = -8, LDY = -10;   // tile size + per-layer offset
function screenPos(t){ return { x: OX + (t.x/2)*TW + t.z*LDX, y: OY + (t.y/2)*TH + t.z*LDY }; }

    Click the top-most tile under the cursor
    
        
    Because higher layers overlap the ones below, a click can land on several stacked tiles. We map the mouse into canvas space (accounting for CSS scaling), then pick the live tile whose rectangle contains the point and that is closest to the viewer — highest layer, and within a layer the one drawn last. Then the free test decides whether that click does anything.
    
        
    function tileAt(px, py){
  let best = null;
  for (const t of tiles){
    if (t.gone) continue;
    const p = screenPos(t);
    if (px >= p.x && px <= p.x+TW && py >= p.y && py <= p.y+TH)
      if (!best || t.z > best.z ||
          (t.z === best.z && (t.y > best.y || (t.y === best.y && t.x > best.x))))
        best = t;                                  // topmost / frontmost wins
  }
  return best;
}

    The move: match, then check the board
    
        
    One click handler ties it together. Ignore blocked tiles, start the timer on the first real move, clear a matching pair, then run the two questions in order: is the board empty (you win), or is there no move left (deadlock, offer a shuffle)? A light requestAnimationFrame only ticks the clock — the board itself is event-driven and redraws on demand.
    
        
    function onClick(t){
