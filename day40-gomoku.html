⚫ Gomoku — Drop Stones on a 15×15 Board and Race to Line Up Five in a Row, in Any Direction, Then Let a Greedy Computer Block and Extend

Five in a row wins — the oldest chase there is.

Gomoku is played on the crossings of a 15×15 board. Two players take turns dropping a stone on any empty point — Black moves first — each trying to line up five of their own colour in an unbroken row. Click an empty intersection to place your stone and the turn passes to your opponent. A winning line counts in any of the four directions — horizontal, vertical, or either diagonal — and the instant one side completes five, that line lights up green and the game is over; no move lands after that. Flip to 🤖 vs Computer and the white stones are played by a small greedy AI that scores every empty point for how strong a line it would build and how badly it needs to block yours, then plays the best of the two. ↩ Undo lifts the last stone (and the computer's reply), and ⚫ New game clears the board.

There are no images and no game library here — the board is a 15×15 grid of single characters, a dot for an empty point and b or w for a stone. Detecting a win never rescans the whole board: from the one point that just changed, it counts identical stones outward along each of the four directions, adds the two arms to the stone itself, and declares a win the moment any of those lines reaches five. That same outward count — scored for how long a run it forms and how many open ends it still has — is all the “intelligence” the optional opponent needs: it values every empty point twice, once as its own attack and once as the block it denies you, and simply plays the highest total. 🔨 Build it yourself

    
The board is a 15×15 grid of single characters — a dot for empty, b or w for a stone. Two players alternate, Black first. The whole game turns on one idea: after each stone lands, look outward from that one point along the four directions, count the identical stones on both sides, and if the stone plus its two arms reach five, that colour has won — collect the line and freeze the board. The optional opponent reuses the very same outward count as a score: every empty point is worth what it builds for the AI plus what it denies you, and it plays the best total. About 120 lines of vanilla JavaScript, no libraries. Copy each block in order.

    The board is 225 characters
    
        
    No sprites, no objects — the whole position is fifteen strings, split into a 15×15 array of characters. A dot is an empty point, b is a Black stone and w is White. Black always plays first. One tiny helper swaps the colour whose turn it is, and every rule below reads from this same grid.
    
        
    const SIZE  = 15;
const START = Array.from({ length: SIZE }, () => ".".repeat(SIZE));
const other = c => c === "b" ? "w" : "b";       // swap the side to move
let board = START.map(row => row.split(""));     // 15×15 of "." / "b" / "w"
let turn  = "b";                                 // Black moves first
let moves = [];                                  // history: {r,c,col}

  2. 
        
    2
    Four directions and a bounds check
    
        
    A line can run four ways: across, down, and the two diagonals. We do not list all eight compass directions — each of the four already covers both of its ends when we walk it forwards and backwards. One inB helper keeps every coordinate on the 15×15 grid so nothing else has to think about the edges.
    
        
    const inB  = (r,c) => r>=0 && r<SIZE && c>=0 && c<SIZE;
const DIRS = [
  [0, 1],   // →  horizontal
  [1, 0],   // ↓  vertical
  [1, 1],   // ↘  main diagonal
  [1,-1]    // ↙  anti-diagonal
];

  3. 
        
    3
    Drop a stone
    
        
    Placing a stone is the only way the board ever changes. Reject the click if the point is already taken or the game is finished; otherwise write the colour, remember the move for undo and the move list, and mark it as the last move so the board can ring it. The win / draw / turn-switch decision follows in the next steps.
    
        
    function commit(r, c){
  if (gameOver || board[r][c] !== ".") return;   // finished or occupied
  board[r][c] = turn;                             // drop the stone
  moves.push({ r, c, col: turn });                // for undo + move list
  lastMove = { r, c };                            // ring the newest stone
  finish(r, c);                                   // win? draw? else switch turn
}
    
      
    4. 

      
  4. 
        
    4
    Count outward from the last stone
    
        
    This is the heart of Gomoku. You never rescan the whole board — only the point that just changed can complete a line. From it, step one direction and keep counting while the stones stay the same colour; that gives the length of one arm. A full line through the point is the stone itself, plus the arm going one way, plus the arm going the other way.
    
        
    function runLength(bd, r, c, dr, dc, col){
  let n = 0, rr = r + dr, cc = c + dc;
  while (inB(rr,cc) && bd[rr][cc] === col){ n++; rr += dr; cc += dc; }
  return n;                                        // same-colour stones one way
}
// line through (r,c) in a direction =
//   1  +  runLength(+dr,+dc)  +  runLength(-dr,-dc)

    5
    The four-direction win scan
    
        
    Now run that outward count once per direction. Walk forward collecting matching stones, then walk backward collecting the rest, and you have the whole unbroken line through the point. If any of the four lines is five stones or longer, that colour has won — and because we kept the coordinates, we return the exact line to light up on the board.
    
        
    function winLineFrom(bd, r, c){
  const col = bd[r][c]; if (col === ".") return null;
  for (const [dr,dc] of DIRS){
    const line = [[r,c]];
    let rr=r+dr, cc=c+dc;                                     // walk forward
    while (inB(rr,cc) && bd[rr][cc]===col){ line.push([rr,cc]);    rr+=dr; cc+=dc; }
    rr=r-dr; cc=c-dc;                                         // walk backward
    while (inB(rr,cc) && bd[rr][cc]===col){ line.unshift([rr,cc]); rr-=dr; cc-=dc; }
    if (line.length >= 5) return line;                        // five (or more) — win!
  }
  return null;
}

  6. 
        
    6
    Win, draw, or play on
    
        
    After each stone, three outcomes are possible. If the win scan finds a line, freeze the board, store the line to highlight it, and stop accepting moves. If not but all 225 points are full, it is a draw. Otherwise swap the side to move and continue. This is the whole game loop.
    
        
    function finish(r, c){
  const line = winLineFrom(board, r, c);
  if (line){                                       // five in a row
    gameOver = true; winLine = line;
    status = (turn === "b" ? "Black" : "White") + " wins — five in a row!";
    return;
  }
  if (board.every(row => row.every(x => x !== "."))){
    gameOver = true; status = "Board full — draw"; return;
  }
  turn = other(turn);                              // nobody won — keep playing
  status = (turn === "b" ? "Black" : "White") + " to move";
}
    
      
    7. 

      
  7. 
        
    7
    Undo lifts the last stone
    
        
    Because every move was pushed onto moves, undo is just a pop: take the last move off the list and clear that point back to a dot. Playing the computer, one human move triggered one machine reply, so undo lifts both — leaving it your turn again. Re-derive whose turn it is straight from how many stones remain.
    
        
    function undo(){
  if (!moves.length) return;
  const m = moves.pop(); board[m.r][m.c] = ".";        // lift the newest stone
  if (vsAI && moves.length % 2 === 1){                  // also lift the AI reply
    const a = moves.pop(); board[a.r][a.c] = ".";
  }
  gameOver = false; winLine = null;
  lastMove = moves.length ? moves[moves.length-1] : null;
  turn = moves.length % 2 === 0 ? "b" : "w";           // Black on even counts
}

  8. 
        
    8
    Score one empty point
    
        
    The optional AI reuses the outward count as a valuation. For a colour played at a point, measure the run it would join in each direction and how many open ends that run still has — an open end is what lets a line grow to five. A tiny table turns (length, open-ends) into a score: a completed five is enormous, an open four is nearly as good, an open three is a real threat.
    
        
    function patternScore(count, open){
  if (count >= 5)  return 1e7;                              // makes five
  if (count === 4) return open === 2 ? 1e6 : open ? 1e4 : 0; // open four vs four
  if (count === 3) return open === 2 ? 1e3 : open ? 1e2 : 0; // open three
  if (count === 2) return open === 2 ? 1e2 : open ? 10  : 0;
  return open ? 1 : 0;
}
function scorePoint(bd, r, c, col){               // value of col playing at (r,c)
  let total = 0;
  for (const [dr,dc] of DIRS){
    let f=0, rr=r+dr, cc=c+dc;                     // run forward
    while (inB(rr,cc) && bd[rr][cc]===col){ f++; rr+=dr; cc+=dc; }
    const fOpen = inB(rr,cc) && bd[rr][cc]===".";  // is that end still open?
    let b=0; rr=r-dr; cc=c-dc;                     // run backward
    while (inB(rr,cc) && bd[rr][cc]===col){ b++; rr-=dr; cc-=dc; }
    const bOpen = inB(rr,cc) && bd[rr][cc]===".";
    total += patternScore(1 + f + b, (fOpen?1:0) + (bOpen?1:0));
  }
  return total;
}

    9
    The greedy AI: block and extend at once
    
        
    Now the opponent is one line. Look only at empty points near existing stones (an empty board just takes the centre), and value each one twice: what it builds for the AI, and what it denies you if the AI takes it first. Adding the two — with a hair more weight on attack — means a point that both extends the AI's threat and blocks yours rises to the top. Its own winning five outscores everything, so it takes a win when it sees one.
    
        
    function candidates(bd){                            // empty points within 2 of a stone
  const seen = new Set(), out = []; let any = false;
  for (let r=0;r<SIZE;r++) for (let c=0;c<SIZE;c++) if (bd[r][c] !== "."){
    any = true;
    for (let dr=-2;dr<=2;dr++) for (let dc=-2;dc<=2;dc++){
      const rr=r+dr, cc=c+dc, k=rr+","+cc;
      if (inB(rr,cc) && bd[rr][cc]==="." && !seen.has(k)){ seen.add(k); out.push([rr,cc]); }
    }
  }
  return any ? out : [[7,7]];                       // first move → centre
}
function aiMove(){
  let best = null, bestScore = -Infinity;
  for (const [r,c] of candidates(board)){
    const s = scorePoint(board,r,c,ai) * 1.05       // my own threat (slight bias)
            + scorePoint(board,r,c,human);          // + the block I deny you
    if (s > bestScore){ bestScore = s; best = [r,c]; }
  }
  if (best) commit(best[0], best[1]);
}

    Click to place, and paint the board
    
        
    One click handler ties it together. Ignore clicks on taken points or after the game ends, and — playing the computer — ignore them when it is not your turn. Otherwise place the stone, redraw, and if the machine is on and the game is still live, let it reply after a short beat. Rendering is a loop over 225 cells that drops a black or white disc on each stone, rings the last move, and glows every cell of the winning line.
    
        
    boardEl.addEventListener("click", e => {
  const cell = e.target.closest(".cell"); if (!cell || gameOver) return;
  const r = +cell.dataset.r, c = +cell.dataset.c;
  if (board[r][c] !== ".") return;                  // occupied
  if (vsAI && turn !== human) return;               // not your turn
  commit(r, c); render();
  if (vsAI && !gameOver) setTimeout(() => { aiMove(); render(); }, 250);
});
// render() loops 225 cells: draw .stone.b / .stone.w on each occupied point,
// ring the last move, and add .win to every cell in winLine.
