🚢 Battleship — Hide Your Fleet, Fire on the Grid, and Hunt-and-Target the Enemy to the Bottom

Play Battleship against the AI.

Hide your five ships, then take turns firing on the enemy's 10×10 grid until one fleet is sunk. Place your fleet on the left board — click a cell to drop the current ship, Rotate to turn it, or let Auto-place scatter them; ships may not overlap or hang off the edge. The AI hides its own five ships the same way. Then the battle: click any cell on the right (enemy waters) to fire — a shot is a miss (water) or a hit (💥), and when every cell of a ship is hit it is sunk. Sink all five enemy ships to win before the AI sinks yours. The interesting part is the opponent. Set it to Random and it fires blindly, wasting most shots on empty water. Set it to Hunt & Target and it plays like a person: it searches on a checkerboard parity pattern — since the smallest ship is two cells long, hitting every other square is enough to find any ship — and the instant it lands a hit it switches to target mode, firing the neighbouring cells; once two hits line up it locks the axis and finishes the ship. Watch the shot counter: the smart AI needs far fewer shots.

Everything here is real Battleship. Each fleet is one 10×10 array of small integers — 0 for open water, otherwise the id of the ship that occupies the cell — plus a parallel shots grid that records what has been fired where (unknown, miss, or hit). Placement is validated cell by cell: a ship is legal only if every square it would cover is on the board and still empty, which is exactly what rules out overlaps and off-board pieces. A shot looks up the target cell, marks the shots grid, and if it struck a ship increments that ship's hit count; when the count reaches the ship's length the ship is sunk, and when all five are sunk the fleet is lost. The Random AI just picks any untried cell. The Hunt & Target AI keeps a little memory: while hunting it fires on a checkerboard parity so no ship of length two or more can hide between its shots; the moment it hits it enters target mode and queues the hit's four orthogonal neighbours; once it has two hits in a row or column it throws away the off-axis candidates and fires only along that line until the ship sinks, then clears its memory and hunts again. All vanilla JavaScript, offline, no images or libraries.

🔨 Build it yourself

    
Two 10×10 integer grids and a five-ship fleet, a random placer that validates every cell, a fire function that resolves hit/miss and detects a sunk ship, a naive random AI as the baseline, then the real opponent: parity hunting, a target queue that locks onto a ship's axis, and a probability-density heat map. About 150 lines of vanilla JS — copy each block in order.

    
    

      
  1. 
        
    1
    The board and the fleet
    
        
    Each side owns a 10×10 grid of integers — 0 is open water, any other number is the id of the ship sitting on that cell — plus a parallel shots grid recording what has been fired at it. The standard fleet is five ships of lengths 5, 4, 3, 3 and 2, so exactly 17 cells are occupied out of 100.
    
        
    const SIZE = 10;
const FLEET = [                              // the classic five ships
  { name:"Carrier",    size:5 },
  { name:"Battleship", size:4 },
  { name:"Cruiser",    size:3 },
  { name:"Submarine",  size:3 },
  { name:"Destroyer",  size:2 },
];                                           // 5+4+3+3+2 = 17 cells out of 100
const inB = (r,c) => r>=0 && r<SIZE && c>=0 && c<SIZE;
const grid0 = () => Array.from({length:SIZE}, () => Array(SIZE).fill(0));
// a "state" is { board, ships, shots }:
//   board[r][c] = 0 water, else ship id (index+1)
//   shots[r][c] = 0 unknown, 1 miss, 2 hit

    2
    Random placement, validated cell by cell
    
        
    A ship is a run of cells, horizontal or vertical, from an anchor square. It is legal only if every one of those cells is on the board and still empty — that single rule is what forbids overlaps and off-board ships. To place the fleet, keep drawing a random square and orientation until the ship fits, then stamp its id onto the board.
    
        
    function shipCells(r, c, size, horizontal){
  const cells = [];
  for (let i=0;i<size;i++) cells.push(horizontal ? [r,c+i] : [r+i,c]);
  return cells;
}
// legal ONLY if every cell is on-board AND empty — no overlap, no overhang
function canPlace(board, cells){
  return cells.every(([r,c]) => inB(r,c) && board[r][c] === 0);
}
function randomFleet(){
  const board = grid0(), ships = [];
  FLEET.forEach((spec, idx) => {
    for (;;){                                  // retry until it fits
      const horizontal = Math.random() < 0.5;
      const r = (Math.random()*SIZE)|0, c = (Math.random()*SIZE)|0;
      const cells = shipCells(r, c, spec.size, horizontal);
      if (!canPlace(board, cells)) continue;
      cells.forEach(([rr,cc]) => board[rr][cc] = idx+1);   // stamp the id
      ships.push({ name:spec.name, size:spec.size, cells, hits:0, sunk:false });
      break;
    }
  });
  return { board, ships, shots: grid0() };
}

    3
    Firing a shot: hit or miss
    
        
    A shot names a cell. Refuse a repeat by checking the shots grid first. Look up the board: water is a miss (mark 1); a ship id is a hit (mark 2) and we bump that ship's hit counter. Everything the rest of the game needs — win detection, the AI's memory — reads out of these two grids.
    
        
    function fire(state, r, c){
  if (state.shots[r][c] !== 0) return null;    // already fired here
  const id = state.board[r][c];
  if (id === 0){ state.shots[r][c] = 1; return { result:"miss", r, c }; }
  state.shots[r][c] = 2;                        // a hit
  const ship = state.ships[id-1];
  ship.hits++;
  if (ship.hits === ship.size){ ship.sunk = true; return { result:"sunk", ship, r, c }; }
  return { result:"hit", ship, r, c };
}

  4. 
        
    4
    Sunk detection and winning
    
        
    Because each hit increments the ship's counter, a ship is sunk the moment its hit count equals its length — no scanning required. A fleet is lost when every ship is sunk. That is the whole win condition: the first player to sink all five enemy ships wins.
    
        
    // sunk is decided in fire(): ship.hits === ship.size  →  ship.sunk = true
function allSunk(state){ return state.ships.every(s => s.sunk); }
function shipsAfloat(state){ return state.ships.filter(s => !s.sunk).length; }
// after each shot:
//   if (allSunk(enemy)) you win;  if (allSunk(me)) the AI wins.

  5. 
        
    5
    The naive AI: pure random
    
        
    The baseline opponent has no memory at all: it lists every cell it has not fired at and picks one at random. It will win eventually, but with 17 ship cells hidden in 100 it wastes most of its shots on water and typically needs the high-90s worth of shots to clear the board. This is the bar the smart AI must beat.
    
        
    function untriedCells(shots){
  const out = [];
  for (let r=0;r<SIZE;r++) for (let c=0;c<SIZE;c++)
    if (shots[r][c] === 0) out.push([r,c]);
  return out;
}
function randomShot(shots){
  const pool = untriedCells(shots);
  return pool[(Math.random()*pool.length)|0];   // blind — no learning
}
    
      
    6. 

      
  6. 
        
    6
    Parity hunting: skip every other cell
    
        
    The first real idea: while searching for a ship you have not touched yet, you never need to fire on two adjacent cells. Every ship is at least two cells long, so it must cover one dark and one light square of a checkerboard. Fire only on cells where (r+c) is even and you still cannot miss a ship — while halving the squares you must consider.
    
        
    // HUNT: fire only on one colour of the checkerboard.
// every ship (length >= 2) covers both colours, so this can't miss one.
function parityShot(shots, parity=0){
  const par = [], all = [];
  for (let r=0;r<SIZE;r++) for (let c=0;c<SIZE;c++)
    if (shots[r][c] === 0){
      all.push([r,c]);
      if (((r+c) & 1) === parity) par.push([r,c]);   // the checkerboard
    }
  const pool = par.length ? par : all;
  return pool[(Math.random()*pool.length)|0];
}

  7. 
        
    7
    Target mode: queue neighbours, then lock the axis
    
        
    The moment a hunt shot connects, switch to target mode. Remember the hits belonging to the current ship. With one hit, the ship could run in any direction, so queue all four orthogonal neighbours. With two hits, you know the ship's axis — throw away the off-axis candidates and fire only at the two open ends of that line until the ship sinks. Then clear the memory and hunt again.
    
        
    const ORTHO = [[-1,0],[1,0],[0,-1],[0,1]];
// rebuild the list of cells to try from the current ship's known hits
function targetQueue(hits){
  if (hits.length === 1){                         // unknown axis → try all 4
    const [r,c] = hits[0];
    return ORTHO.map(([dr,dc]) => [r+dr, c+dc]);
  }
  const sameRow = hits.every(h => h[0] === hits[0][0]);
  if (sameRow){                                   // horizontal → extend the row
    const r = hits[0][0], cs = hits.map(h => h[1]).sort((a,b)=>a-b);
    return [[r, cs[0]-1], [r, cs[cs.length-1]+1]];
  }
  const c = hits[0][1], rs = hits.map(h => h[0]).sort((a,b)=>a-b);
  return [[rs[0]-1, c], [rs[rs.length-1]+1, c]];   // vertical → extend the column
}
function smartShot(ai, shots){
  if (ai.hits.length){                            // TARGET: follow the ship
    for (const [r,c] of targetQueue(ai.hits))
      if (inB(r,c) && shots[r][c] === 0) return [r,c];
  }
  return parityShot(shots, ai.parity);            // HUNT: checkerboard search
}
// after each AI shot, feed the result back:
function observe(ai, r, c, res){
  if (res.result === "hit")  ai.hits.push([r,c]);        // remember it
  if (res.result === "sunk") ai.hits = [];               // done — hunt anew
}

  8. 
        
    8
    Probability density: where do ships fit best?
    
        
    Parity is a cheap stand-in for the strongest idea: for every ship still afloat, slide it into every legal position on the board (any run that avoids the known misses) and add one to each cell it would cover. The result is a heat map — the cell counted the most times is where a remaining ship can fit the most ways, and it is the mathematically best square to fire. This heuristic naturally concentrates fire in open space and even reproduces the parity pattern on its own.
    
        
    // count, for every remaining ship and every legal position, how many
// ways cover each cell. The hottest untried cell is the best shot.
function densityShot(shots, remainingSizes){
  const heat = grid0();
  for (const size of remainingSizes){
    for (let r=0;r<SIZE;r++) for (let c=0;c<SIZE;c++){
      for (const horiz of [true,false]){
        const cells = shipCells(r, c, size, horiz);
        if (!cells.every(([y,x]) => inB(y,x) && shots[y][x] !== 1)) continue; // clears misses
        cells.forEach(([y,x]) => heat[y][x]++);
      }
    }
  }
  let best = null, bestV = -1;
  for (let r=0;r<SIZE;r++) for (let c=0;c<SIZE;c++)
    if (shots[r][c] === 0 && heat[r][c] > bestV){ bestV = heat[r][c]; best = [r,c]; }
  return best;                                     // most ways a ship still fits
}

    9
    Wire up the turn loop
    
        
    Each side owns a state; the AI owns a small brain (hits, parity). You fire at the enemy state, check for a win, then the AI fires at your state through smartShot and feeds the result to observe. Alternate until one fleet is gone. Rendering is just a readout of the two grids.
    
        
    let me    = randomFleet();          // your hidden fleet (AI fires here)
let enemy = randomFleet();          // the AI's hidden fleet (you fire here)
let ai    = { hits:[], parity:0, mode:"smart" };

function playerFire(r, c){
  const res = fire(enemy, r, c);
  if (!res) return;                 // ignore repeats
  render();
  if (allSunk(enemy)) return win("you");
  aiTurn();
}
function aiTurn(){
  const [r,c] = ai.mode === "random" ? randomShot(me.shots)
                                     : smartShot(ai, me.shots);
  const res = fire(me, r, c);
  observe(ai, r, c, res);           // update hunt/target memory
  render();
  if (allSunk(me)) return win("ai");
}
