🔢 15-Puzzle — Slide the Tiles Into Order

Slide the numbered tiles around the one empty square until they read 1 to 15 in order.

There are fifteen tiles and one blank gap in a 4×4 frame. Click any tile that sits directly next to the gap — above, below, left, or right — and it slides into the empty space. You can also use the arrow keys: an arrow slides the neighbouring tile into the blank in that direction. Keep sliding tiles around until they line up 1, 2, 3 … 15 with the gap back in the bottom-right corner. The move counter and timer track how efficient you are; beat your best on the next round.

The Shuffle button never deals an impossible board: it starts from the solved layout and makes hundreds of real, legal slides at random, so the scramble is always reachable — and therefore always solvable — by definition. (Exactly half of all 16! tile arrangements can't be solved; the inversion-parity theorem on the UNDERSTAND tab explains why.) A tile that's already sitting in its correct home glows a lighter green so you can see your progress. Everything runs offline in plain vanilla JS — both clicking tiles and the arrow keys work.

🔨 Build it yourself

    
A board held as a flat array of 16 cells (numbers 1–15 plus a 0 for the blank), absolute-positioned tiles that slide with a CSS transition, a click handler that swaps an adjacent tile with the blank, arrow keys that pull a tile into the gap, a shuffle that does many random legal moves so the result is always solvable, a win check that compares the array to its sorted order, and a move counter plus a timer. ~90 lines of vanilla JS — copy each block in order.

    
    

      
  1. 
        
    1
    Model the board as a flat array
    
        
    The whole puzzle is one array of 16 numbers: 1 through 15 for the tiles and a 0 standing in for the blank. Index i in the array maps to grid position row Math.floor(i/4), column i%4. The solved state is simply [1,2,3,…,15,0]. Keeping it flat (not a 2D array) makes swapping, shuffling, and win-checking one-liners — we only convert an index to row/column when we need to draw a tile.
    
        
    
          
    const SIZE = 4, N = SIZE * SIZE;            // 4×4 = 16 cells
let cells = [];                            // cells[i] = tile number, 0 = blank
const SOLVED = [...Array(N - 1)].map((_, i) => i + 1).concat(0); // [1..15, 0]

function rowOf(i){ return Math.floor(i / SIZE); }
function colOf(i){ return i % SIZE; }
function blankIndex(){ return cells.indexOf(0); }   // where the gap is
    
          
        
    
      
    2. 

      
  2. 
        
    2
    Render tiles by position
    
        
    Every tile is one absolutely-positioned element whose left and top come from its current index. We build the fifteen tile elements once and keep them keyed by tile number; on each render we just move each tile to the pixel coordinates of wherever its number currently lives in the array. Because the elements keep their identity and only their position changes, the CSS transition animates them smoothly between renders. The blank is nothing — we simply don't draw a tile for the 0.
    
        
    
          
    const GAP = 8, TILE = 80;                   // padding + cell pitch (px)
function render(){
  for (let i = 0; i < N; i++){
    const n = cells[i];
    if (n === 0) continue;                  // blank: draw nothing
    const el = tileEls[n];                  // the element for tile #n
    el.style.left = (GAP + colOf(i) * TILE) + "px";
    el.style.top  = (GAP + rowOf(i) * TILE) + "px";
    el.classList.toggle("locked", n === i + 1);  // glow if already home
  }
}

  3. 
        
    3
    A legal move = swap with the blank
    
        
    A tile can only move if it sits directly next to the blank. "Next to" means the same column one row apart, or the same row one column apart — never diagonal, and never wrapping around an edge. When a tile qualifies, the move is just swapping its array slot with the blank's slot. That single swap is the slide; the render step then animates the tile into the gap. We return whether the move happened so the caller can count it.
    
        
    
          
    function isAdjacent(a, b){                  // are cells a and b neighbours?
  const dr = Math.abs(rowOf(a) - rowOf(b));
  const dc = Math.abs(colOf(a) - colOf(b));
  return dr + dc === 1;                     // exactly one step apart
}
function moveTileAt(i){
  const b = blankIndex();
  if (!isAdjacent(i, b)) return false;     // not next to the gap → ignore
  [cells[i], cells[b]] = [cells[b], cells[i]];   // swap tile ↔ blank
  return true;
}
    
          
        
    
      
    4. 

      
  4. 
        
    4
    Click and arrow-key controls
    
        
    Clicking a tile tries to move that specific tile. Arrow keys work the other way round: they describe which neighbour of the blank slides into it. Pressing ↑ pulls the tile below the gap upward, ↓ pulls the one above downward, and so on — the direction names the motion of the tile, not of the gap. Both paths end at the same tryMove, which moves the tile, re-renders, counts the move, and checks for a win.
    
        
    
          
    function tryMove(i){
  if (moveTileAt(i)){ moves++; render(); checkWin(); }
}
board.addEventListener("click", e => {
  const n = +e.target.dataset.n;           // which tile number was clicked
  if (n) tryMove(cells.indexOf(n));
});
document.addEventListener("keydown", e => {
  const b = blankIndex(), r = rowOf(b), c = colOf(b);
  if (e.key === "ArrowUp"    && r < 3) tryMove(b + SIZE); // tile below slides up
  if (e.key === "ArrowDown"  && r > 0) tryMove(b - SIZE); // tile above slides down
  if (e.key === "ArrowLeft"  && c < 3) tryMove(b + 1);    // tile right slides left
  if (e.key === "ArrowRight" && c > 0) tryMove(b - 1);    // tile left slides right
});
    
          
        
    
      
    5. 

      
  5. 
        
    5
    Shuffle that's always solvable
    
        
    The safe way to scramble is to start from the solved board and make a few hundred real moves at random — repeatedly slide a random neighbour of the blank into it. Because every step is a legal move, the result is guaranteed to be reachable from solved, and therefore solvable. We avoid instantly undoing the previous move so the board genuinely mixes. This sidesteps the inversion-parity check entirely: a randomly shuffled array has a 50% chance of being impossible, but a randomly played board never is.
    
        
    
          
    function shuffle(steps = 300){
  cells = [...SOLVED];
  let prev = -1;
  for (let s = 0; s < steps; s++){
    const b = blankIndex();
    const nbrs = [b - SIZE, b + SIZE, b - 1, b + 1]
      .filter(j => j >= 0 && j < N && isAdjacent(j, b) && j !== prev);
    const pick = nbrs[Math.floor(Math.random() * nbrs.length)];
    [cells[b], cells[pick]] = [cells[pick], cells[b]];   // slide it in
    prev = b;                                            // don't undo next time
  }
  moves = 0; render();
}
    
          
        
    
      
    6. 

      
  6. 
        
    6
    The inversion-parity test (optional)
    
        
    If you'd rather scramble by shuffling the array directly, you must verify it's solvable. Count "inversions": every pair of tiles that appears in the wrong relative order. For a 4-wide board the rule is: the puzzle is solvable when the inversion count plus the row of the blank (counted from the bottom, 1-based) is even. Reject and reshuffle anything that fails this test. It's the mathematical guarantee behind why our move-based shuffle is always safe.
    
        
    
          
    function isSolvable(a){
  const tiles = a.filter(n => n !== 0);     // ignore the blank
  let inv = 0;
  for (let i = 0; i < tiles.length; i++)
    for (let j = i + 1; j < tiles.length; j++)
      if (tiles[i] > tiles[j]) inv++;        // a pair out of order
  const blankRowFromBottom = SIZE - rowOf(a.indexOf(0));   // 1-based, from bottom
  return (inv + blankRowFromBottom) % 2 === 1;   // the 4×4 (even-width) parity rule
}
    
          
        
    
      
    7. 

      
  7. 
        
    7
    Win detection
    
        
    Winning is the simplest check in the whole game: the board is solved when the array equals [1,2,…,15,0]. We compare every cell to its target and, on a match, stop the timer and announce the result with the move count and elapsed time. A best-score in localStorage lets the player chase a personal record across rounds. Because the array fully describes the state, one equality scan is all the win logic we need.
    
        
    
          
    function isSolved(){
  return cells.every((n, i) => n === SOLVED[i]);   // [1..15,0] in order
}
function checkWin(){
  if (!isSolved()) return;
  stopTimer();
  const best = +localStorage.puzzleBest || Infinity;
  if (moves < best) localStorage.puzzleBest = moves;     // new record
  msg.textContent = `🎉 Solved in ${moves} moves · ${fmt(seconds)}`;
}

  8. 
        
    8
    Move counter and timer
    
        
    Two stats make the puzzle a challenge instead of idle fiddling. A counter ticks up once per successful slide. The timer starts on the first move after a shuffle and runs once per second until you solve it, formatting seconds as m:ss. We start lazily — no clock runs while the board sits untouched — and clear the interval the instant the puzzle is solved so the final time freezes. Display both on every render so the numbers stay in sync with the board.
    
        
    
          
    let moves = 0, seconds = 0, timer = null;
function startTimer(){
  if (timer) return;                        // already running
  timer = setInterval(() => { seconds++; timeEl.textContent = fmt(seconds); }, 1000);
}
function stopTimer(){ clearInterval(timer); timer = null; }
function fmt(s){ return Math.floor(s / 60) + ":" + String(s % 60).padStart(2, "0"); }
// call startTimer() inside tryMove on the first move; movesEl shows `moves`
