🌀 Maze — Carve a Perfect Maze with DFS, Then Solve It with BFS

Carve a maze, then watch it get solved.

Every cell starts boxed in by four walls. Generate runs a randomised depth-first search (the recursive-backtracker): it wanders from the top-left corner, knocking down a wall into a random unvisited neighbour, diving as deep as it can, and backtracking when it hits a dead end — carving a perfect maze where every cell is reachable by exactly one route. Solve then runs a breadth-first search from the entrance (top-left) to the exit (bottom-right): it explores outward in rings, and the first time it touches the exit it has found the shortest path, which lights up in emerald. Or grab the white dot and walk it yourself with the arrow keys.

The maze is one 2-D array where each cell stores a bitmask of the walls still standing (four bits: north, east, south, west). Carving is a real stack-based DFS: push the start, keep diving into a random unvisited neighbour and clearing the shared wall bit on both cells, and pop to backtrack when boxed in. It removes exactly cells − 1 walls, which makes the maze a spanning tree — no loops, no islands, one unique path between any two cells. Solving is a real BFS: a FIFO queue, a visited grid, and a parent map so that once the exit is dequeued we walk the parents back to reconstruct the route. Because every corridor step costs the same, BFS is already optimal — no need for Dijkstra or A*. All vanilla JavaScript on a canvas, offline, no images or libraries.

  1. 
        
    1
    The state: each cell is a bitmask of the walls still up
    
        
    The whole maze is a grid of small integers. Each cell uses four bits — one per wall (north, east, south, west). 15 is binary 1111: all four walls present, a solid uncarved cell. Carving a passage just clears a bit. Start every cell at 15.
    
        
    let N = 20;                       // maze is N x N cells
const N_=1, E_=2, S_=4, W_=8;     // one bit per wall
let cells = [];                   // cells[r][c] = bitmask of walls still up
// 15 = 1111 = all four walls present (a solid, uncarved cell)
function reset(){
  cells = Array.from({length:N}, () => Array(N).fill(15));
}
    
      
    2. 

      
  2. 
        
    2
    Four directions — and the wall two cells share
    
        
    Moving between two adjacent cells crosses one wall that both cells own — my north wall is my neighbour's south wall. So each direction carries the bit to clear on this cell (bit) and the matching bit on the neighbour (opp). Clearing both keeps the two cells' views of the wall in sync.
    
        
    // dr,dc = step to the neighbour; bit = wall on THIS cell,
// opp = the matching wall on the NEIGHBOUR (they share it)
const DIRS = [
  {dr:-1, dc:0, bit:N_, opp:S_},   // north
  {dr:0,  dc:1, bit:E_, opp:W_},   // east
  {dr:1,  dc:0, bit:S_, opp:N_},   // south
  {dr:0, dc:-1, bit:W_, opp:E_},   // west
];
const inB = (r,c) => r>=0 && r<N && c>=0 && c<N;
    
      
    3. 

      
  3. 
        
    3
    Carve: randomised DFS (recursive backtracker)
    
        
    Push the start cell, then repeat: look at the top of the stack, pick a random unvisited neighbour, knock down the shared wall on both sides, mark it visited, and push it — diving deeper. When a cell has no unvisited neighbours it's a dead end, so pop and backtrack. It ends after removing exactly N·N−1 walls: a spanning tree, i.e. a perfect maze.
    
        
    function carve(rand){
  reset();
  const seen = Array.from({length:N}, () => Array(N).fill(false));
  const stack = [[0,0]];  seen[0][0] = true;
  while (stack.length){
    const [r,c] = stack[stack.length-1];        // peek the top
    const opts = DIRS.filter(d => {
      const nr=r+d.dr, nc=c+d.dc;
      return inB(nr,nc) && !seen[nr][nc];        // unvisited neighbours
    });
    if (!opts.length){ stack.pop(); continue; }  // dead end -> backtrack
    const d = opts[(rand()*opts.length)|0];      // pick one at random
    const nr=r+d.dr, nc=c+d.dc;
    cells[r][c]   &= ~d.bit;                      // knock down the shared wall
    cells[nr][nc] &= ~d.opp;                      // ...on both sides
    seen[nr][nc] = true;
    stack.push([nr,nc]);                          // dive deeper
  }
  // exactly N*N-1 walls removed => spanning tree => a perfect maze
}
    
      
    4. 

      
  4. 
        
    4
    Draw the walls that are still standing
    
        
    Rendering is a pure function of the array: for each cell, draw only the wall segments whose bit is still set. One cell is S pixels square; its top-left is (c·S, r·S). Test each bit and stroke that edge. That's the entire maze — no sprites, just lines.
    
        
    const ctx = canvas.getContext("2d");
const S = 24;                          // pixel size of one cell
function draw(){
  ctx.clearRect(0,0,canvas.width,canvas.height);
  ctx.strokeStyle = "#e2e8f0"; ctx.lineWidth = 2;
  for (let r=0;r<N;r++) for (let c=0;c<N;c++){
    const x=c*S, y=r*S, w=cells[r][c];
    ctx.beginPath();
    if (w & N_){ ctx.moveTo(x,   y  ); ctx.lineTo(x+S, y  ); } // top
    if (w & E_){ ctx.moveTo(x+S, y  ); ctx.lineTo(x+S, y+S); } // right
    if (w & S_){ ctx.moveTo(x,   y+S); ctx.lineTo(x+S, y+S); } // bottom
    if (w & W_){ ctx.moveTo(x,   y  ); ctx.lineTo(x,   y+S); } // left
    ctx.stroke();
  }
}

  5. 
        
    5
    Solve: breadth-first search with a parent map
    
        
    BFS uses a FIFO queue, so it explores in expanding rings — all cells one step away, then two, and so on. You can move to a neighbour only if the wall between you is already gone (its bit is clear). Record where each cell was reached from in a parent grid. Because every step costs the same, the first time BFS reaches the exit it has done so along a shortest path.
    
        
    function solve(){
  const start=[0,0], goal=[N-1,N-1];
  const seen   = Array.from({length:N}, () => Array(N).fill(false));
  const parent = Array.from({length:N}, () => Array(N).fill(null));
  const queue = [start];  seen[0][0] = true;
  while (queue.length){
    const [r,c] = queue.shift();               // FIFO -> explore in rings
    if (r===goal[0] && c===goal[1]) break;      // reached the exit
    for (const d of DIRS){
      if (cells[r][c] & d.bit) continue;         // wall in the way -> blocked
      const nr=r+d.dr, nc=c+d.dc;
      if (!seen[nr][nc]){
        seen[nr][nc] = true;
        parent[nr][nc] = [r,c];                  // remember how we got here
        queue.push([nr,nc]);
      }
    }
  }
  return parent;
}

  6. 
        
    6
    Reconstruct the shortest path from the parents
    
        
    The parent map is a chain of "who reached me". Start at the exit and hop parent-to-parent until you fall off the start (whose parent is null), then reverse. In a perfect maze this path is the only one, so it is automatically shortest — but the same BFS reconstruction still gives the shortest route the instant a maze has loops.
    
        
    function shortestPath(parent){
  const path = [];
  let cur = [N-1, N-1];                          // start at the exit
  while (cur){ path.push(cur); cur = parent[cur[0]][cur[1]]; }
  return path.reverse();                          // entrance -> exit order
}
// perfect maze => unique path => this IS the shortest path.
// braided maze (with loops) => BFS still returns the shortest.

    Walk it yourself with the arrow keys
    
        
    Give the player a position and move it on each arrow key — but only if the wall in that direction is already down (same bit test the solver uses). It's the same graph, walked by hand instead of by BFS. Reach the bottom-right and you've escaped.
    
        
    let player = [0,0];
addEventListener("keydown", e => {
  const map = { ArrowUp:[-1,0], ArrowRight:[0,1], ArrowDown:[1,0], ArrowLeft:[0,-1] };
  const mv = map[e.key];  if (!mv) return;
  e.preventDefault();
  const [dr,dc] = mv, [r,c] = player;
  const bit = dr===-1?N_ : dr===1?S_ : dc===1?E_ : W_;
  if (cells[r][c] & bit) return;                 // wall -> can't move
  player = [r+dr, c+dc];  draw();
  if (player[0]===N-1 && player[1]===N-1) alert("You reached the exit!");
});

    Wire it up: seed, size, regenerate, solve
    
        
    A tiny seeded generator (mulberry32) makes every maze repeatable: the same seed rebuilds the same maze, so "Regenerate" is just a new number. The grid selector changes N and rebuilds; Solve runs BFS and animates the path. Every rule is a small operation on one 2-D array.
    
        
    function generate(seed){
  carve(mulberry32(seed));   // seeded -> same number rebuilds same maze
  player = [0,0];
  draw();
}
function mulberry32(a){ return function(){ a|=0; a=a+0x6D2B79F5|0;
  let t=Math.imul(a^a>>>15,1|a); t=t+Math.imul(t^t>>>7,61|t)^t;
  return ((t^t>>>14)>>>0)/4294967296; }; }

sizeSel.onchange = e => { N = +e.target.value; generate((Math.random()*1e9)|0); };
solveBtn.onclick = () => { const p = solve(); animatePath(shortestPath(p)); };
newBtn.onclick   = () => generate((Math.random()*1e9)|0);
generate(12345);
