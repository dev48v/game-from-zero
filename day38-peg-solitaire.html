🔘 Peg Solitaire — Jump One Peg Over Another Into an Empty Hole and Whittle the Board Down to a Single Survivor

Thirty-two pegs, one empty hole — leap your way down to a single survivor.

This is the classic English cross board: a 7×7 grid with the four corners cut away, leaving 33 holes in a plus shape. Every hole holds a peg except the one dead in the centre. Your only move is a jump: pick a peg that has an adjacent peg straight beside it and an empty hole straight beyond that peg, and hop over — the jumped peg is lifted off the board. Every jump therefore removes exactly one peg, so you must go from 32 pegs down toward 1. Click a peg to select it and its legal landing holes light up green; click a lit hole to make the leap. The perfect finish is one lone peg sitting in the centre hole — but any single survivor is a win. Jump carelessly and you can strand yourself with pegs scattered and no legal jump left; the board tells you the instant that happens and counts how many you left behind. ↩ Undo walks moves back, and 💡 Hint shows you one jump that still works.

There are no sprites and no physics here — the whole game is a 7×7 grid of three possible states: a wall (the cut-off corners), an empty hole, or a peg. One rule runs everything. A jump is legal exactly when a peg has, in one of the four straight directions, an adjacent peg and then an empty hole immediately beyond it; making the jump sets the starting cell and the jumped cell to empty and the landing cell to a peg, so the count drops by one every time. Legal-move generation is just that same test asked of every peg in all four directions, and a highlight is nothing more than the subset of those moves that start from the peg you clicked. After each jump the board asks two questions in order: is only one peg left — you win, and if it is on the centre it is the textbook solution — or is there no legal jump anywhere, in which case the remaining pegs are stranded and the game is over. An undo stack keeps a snapshot before each leap so you can back out of a bad line. All vanilla JavaScript on one canvas, offline, no images and no libraries.

🔨 Build it yourself

    
The board is a 7×7 grid holding three states — wall, empty, peg — with the four corners walled off to make the cross. One test answers everything: a jump is legal when a peg has an adjacent peg and then an empty hole two cells away in a straight line. Legal-move generation is that test over every peg in four directions; a highlight is the moves that start from your selected peg; a move flips three cells and drops the peg count by one. Add a win-versus-stuck check after every jump, an undo snapshot stack, and a plain canvas draw of holes and pegs. About 140 lines of vanilla JavaScript on one canvas — no images, no libraries. Copy each block in order.

    
    

      
  1. 
        
    1
    The board is a cross carved out of a 7×7 grid
    
        
    There is no clever shape data — just a 7×7 grid where each cell is one of three numbers: a WALL (the four cut-off corners), an EMPTY hole, or a PEG. A cell is off-board precisely when it sits in a corner block, which is true when its row is outside the middle three and its column is outside the middle three. Every hole starts with a peg except the very centre, which starts empty.
    
        
    const WALL = -1, EMPTY = 0, PEG = 1;

// a cell is on the cross unless it is in one of the four 2×2 corners
function onBoard(r, c){
  if (r < 0 || r > 6 || c < 0 || c > 6) return false;
  return !((r < 2 || r > 4) && (c < 2 || c > 4));   // corner = row AND col both outside the middle 3
}

function newBoard(){
  board = [];
  for (let r = 0; r < 7; r++){
    const row = [];
    for (let c = 0; c < 7; c++)
      row.push(!onBoard(r, c) ? WALL : (r === 3 && c === 3 ? EMPTY : PEG));   // centre empty
    board.push(row);
  }
  pegs = 32;                                    // 33 holes − 1 empty centre
}
    
      
    2. 

      
  2. 
        
    2
    Four directions, and a safe read of any cell
    
        
    A jump only ever goes straight — up, down, left, or right. A tiny at() helper reads any coordinate and returns WALL for anything off the grid, so the rest of the code never has to guard array bounds. Because empties and pegs only ever exist on valid cells, one clean read of at() is all the geometry we need.
    
        
    const DIRS = [[-1, 0], [1, 0], [0, -1], [0, 1]];   // up, down, left, right

function at(r, c){
  return (r >= 0 && r < 7 && c >= 0 && c < 7) ? board[r][c] : WALL;
}
    
      
    3. 

      
  3. 
        
    3
    The jump rule: peg, peg, empty — in a line
    
        
    This is the whole game in one function. From a peg at (r,c), step one cell in a direction to the over cell and two cells to the to cell. The jump is legal only when the over cell holds a peg and the to cell is empty. movesFrom collects every legal jump that starts at one peg — that is exactly the set of holes we light up when you select it.
    
        
    function movesFrom(r, c){
  const moves = [];
  if (at(r, c) !== PEG) return moves;
  for (const [dr, dc] of DIRS){
    const mr = r + dr,   mc = c + dc;       // the peg we jump OVER
    const tr = r + 2*dr, tc = c + 2*dc;     // the empty hole we land IN
    if (at(mr, mc) === PEG && at(tr, tc) === EMPTY)
      moves.push({ r, c, mr, mc, tr, tc }); // a legal jump
  }
  return moves;
}
    
      
    4. 

      
  4. 
        
    4
    Every legal move on the whole board
    
        
    Ask movesFrom of every peg and concatenate the answers, and you have every jump available right now. This one list powers three different things: the hint button picks one from it, the "movable" highlight is the set of pegs that appear as a starting cell, and the stuck check is simply whether this list is empty.
    
        
    function legalMoves(){
  const all = [];
  for (let r = 0; r < 7; r++)
    for (let c = 0; c < 7; c++)
      if (board[r][c] === PEG)
        for (const m of movesFrom(r, c)) all.push(m);
  return all;
}

  5. 
        
    5
    Select a peg, and its landing holes light up
    
        
    Clicking a peg selects it and, from movesFrom, we know exactly which empty holes are legal targets. We stash those target coordinates in a set so the drawing code can ring them in green. Clicking the same peg again clears the selection; clicking a different peg moves it.
    
        
    function selectPeg(r, c){
  if (selected && selected.r === r && selected.c === c){ selected = null; return; }
  selected = { r, c };
  const targets = movesFrom(r, c);          // the holes this peg may leap into
  if (targets.length === 0) msg("That peg has no jump — pick another.", "warn");
  else                      msg("Now click a highlighted hole to jump.");
}
    
      
    6. 

      
  6. 
        
    6
    Making the leap flips exactly three cells
    
        
    A jump is pure bookkeeping. The starting cell becomes empty (the peg left), the jumped-over cell becomes empty (that peg is removed), and the landing cell becomes a peg (where it arrived). The peg count falls by one and the move counter ticks up. Because a jump always deletes one peg, the number left is 32 − moves — you can watch yourself closing in on a single survivor.
    
        
    function doJump(m){
  board[m.r][m.c]   = EMPTY;    // peg leaves its hole
  board[m.mr][m.mc] = EMPTY;    // jumped peg is removed
  board[m.tr][m.tc] = PEG;      // peg lands two cells over
  pegs--; moveCount++;
}
    
      
    7. 

      
  7. 
        
    7
    Win or stuck: two questions after every jump
    
        
    The end of every turn asks two things in order. If only one peg remains you have won — and if that last peg is sitting on the centre hole it is the textbook solution, the goal of the classic puzzle. Otherwise, if legalMoves() comes back empty, the remaining pegs are stranded with no jump possible, so the game is over and we report how many you left behind.
    
        
    function afterMove(){
  if (pegs === 1){
    won = true;
    const centred = board[3][3] === PEG;
    msg(centred ? "🎯 Perfect — one peg, dead centre!" : "🎉 Solved to a single peg!", "win");
    recordBest();
  } else if (legalMoves().length === 0){
    won = true;
    msg("No jumps left — " + pegs + " pegs stranded. New game or undo.", "warn");
    recordBest();
  } else {
    msg("Nice jump — keep hunting for pairs.");
  }
}

  8. 
        
    8
    Undo is just a stack of snapshots
    
        
    Peg solitaire is unforgiving, so undo matters. Before every jump we push a deep copy of the board plus the two counters onto a stack. Undo pops the last snapshot, restores it, and clears the game-over flag — letting you back out of the move that painted you into a corner and try a different line.
    
        
    function pushUndo(){
  undoStack.push({ b: board.map(row => row.slice()), pegs, moveCount });
}

function undo(){
  if (!undoStack.length) return;
  const s = undoStack.pop();
  board = s.b.map(row => row.slice());
  pegs = s.pegs; moveCount = s.moveCount;
  won = false; selected = null;
}

  9. 
        
    9
    Draw holes and pegs on one canvas
    
        
    Each grid cell maps to a pixel centre by simple multiplication. For every on-board cell we sink a dark recessed hole; if it holds a peg we draw a domed disc with a radial gradient. The selected peg gets a white ring, the legal landing holes get a green ring, and — when the toggle is on — every peg that can still jump is outlined. No sprites, all shapes.
    
        
    const CELL = 60, PAD = 20;
function centre(r, c){ return { x: PAD + c*CELL + CELL/2, y: PAD + r*CELL + CELL/2 }; }

function drawPeg(x, y){
  const rad = CELL * 0.30;
  const g = ctx.createRadialGradient(x - rad*0.3, y - rad*0.4, rad*0.2, x, y, rad);
  g.addColorStop(0, "#fcd34d"); g.addColorStop(1, "#c2620a");   // domed amber peg
  ctx.beginPath(); ctx.arc(x, y, rad, 0, 7); ctx.fillStyle = g; ctx.fill();
  ctx.lineWidth = 2; ctx.strokeStyle = "#7c3d06"; ctx.stroke();
}

    10
    One click handler ties it together
    
        
    A click maps to a grid cell. Land on a peg and you select it; land on an empty hole while a peg is selected and we look up whether that hole is a legal target for it — if so we snapshot, jump, and run the win-or-stuck check. Everything else is a gentle nudge in the message line. The board is entirely event-driven and only redraws when something changes.
    
        
    function onCell(r, c){
  if (won || !onBoard(r, c)) return;
  if (board[r][c] === PEG) return selectPeg(r, c);          // pick a peg
  if (board[r][c] === EMPTY && selected){
    const m = movesFrom(selected.r, selected.c)
                .find(x => x.tr === r && x.tc === c);        // is this a legal landing?
    if (m){ pushUndo(); doJump(m); selected = null; afterMove(); }
    else   msg("Not a legal landing — jump straight over one peg.", "warn");
  }
}
